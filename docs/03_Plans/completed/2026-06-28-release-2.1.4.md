# Release 2.1.4 — tag delete-eligible global IPAM

**Date:** 2026-06-28

## Goal
Cut 2.1.4 bundling the delete-eligible IPAM tagging feature (see
`2026-06-28-tag-delete-eligible-ipam.md`).

## Constraints
- Drop-in from 2.1.3; no schema/data change. Tag-only — never deletes.

## Touched Surfaces
- Version bump: `pyproject.toml`, `forward_netbox/__init__.py`, the 3 README
  tables, `CHANGELOG.md`.
- Feature: delete-eligible IPAM tagging — see `2026-06-28-tag-delete-eligible-ipam.md`.

## Approach
Standard release: prepare bumps version + README + CHANGELOG; verify (lint,
harness, sensitive, build, full suite); publish FF main + tag + GitHub release + PyPI.

## Validation
Full suite (19 scope tests + rest) on NetBox 4.6.3. Lint/harness/sensitive. PR CI
green before merge; main CI green after. No `.nqe` change this release, so no CustomerOrg
org query republish needed.

## Rollback
Revert the release commit; `pip install forward-netbox==2.1.3`.

## Decision Log
- Bundled the feature with the version bump in one release branch, matching the
  2.1.1/2.1.2/2.1.3 single-branch release pattern.

## Bundled changes
- The "Prune orphans" Scope Reconciliation page gains a "Tag delete-eligible
  IPAM" action: it audits network-global IPAM (prefixes, VLANs, VRFs) against the
  sync's latest Forward fetch and stamps objects Forward no longer reports with a
  self-healing `forward-delete-eligible` tag for manual review. Tag-only — never
  deletes. The tag set is reconciled every run (an object that returns to Forward
  is auto-untagged), and a model whose Forward fetch comes back empty is skipped
  so a broken fetch cannot flag everything. New `forward_tag_delete_eligible`
  management command. Drop-in from `2.1.3`.
