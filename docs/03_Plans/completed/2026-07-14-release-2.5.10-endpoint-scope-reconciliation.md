# Release 2.5.10 - endpoint-aware scope reconciliation

Status: implementation, aggregate CI, Bay-safe field validation, query
republication, and release-readiness gates passed. Package/tag rollout remains.

## Goal

Ship the reconciliation fix that keeps imported SNMP endpoints, including
Avocent console servers, out of the false out-of-scope orphan set, and repair
the live DLM CVE/Vulnerability query syntax exposed by 2.5.9 query-path
publishing.
The release also prevents DLM SoftwareVersion rows from being created without a
DeviceSoftware association when the Forward source contains devices outside the
NetBox inventory.
It also keeps SNMP endpoint imports inside configured include-tag scope, carries
matching endpoint tags into NetBox, and prevents volatile console-server
software/build strings from creating DeviceType identities.

## Constraints

- No database migration. The DLM CVE, Vulnerability, DeviceSoftware, and active
  device query sources require republishing.
- Endpoint membership must match sync-time endpoint import rules.
- A failed endpoint scope probe must not produce a destructive reconciliation
  result.
- Do not claim a DLM runtime fix without a completed customer preview payload or
  equivalent live query evidence.
- Do not stage or modify the operator's local `development/.env` change.
- Package/tag rollout remains an explicit follow-up after local and GitHub
  validation. Four affected validation-org query paths were repaired during
  live verification: CVE, Vulnerability, DeviceSoftware, and the active
  alias-aware Device map.

## Approach

1. Retain eligible endpoint names in reconciliation using the same SNMP-output,
   include-tag, and exclude-tag predicates as sync-time endpoint resolution.
2. Keep modeled-device names separate for the empty-scope prune guard.
3. Surface endpoint counts in reconciliation output and cover the library, CLI,
   and rendered UI paths.
4. Execute the published DLM query paths against the matching Forward dataset,
   repair the invalid stacked `@primaryKey`/parameterized `@query` syntax, and
   republish only the affected CVE, Vulnerability, and DeviceSoftware paths.
5. Bump release metadata to 2.5.10 and run the release validation gate.
6. Require `pyzipper` 0.4.0+ and cover the patched small-entry AES archive
   behavior after the release dependency audit identified `PYSEC-2026-3044`.
7. Replace the stale architecture-audit wrapper for the management command
   removed in 2.0 with an isolated-safe focused contract test gate.
8. Make DeviceSoftware the authority for SoftwareVersion creation, carry the
   lifecycle fields on its device-scoped query, and make the standalone catalog
   map enrichment-only. Vulnerability imports ensure the same DeviceSoftware
   link when that map is used independently.
9. Classify Opengear and Avocent endpoints as console servers, derive stable
   hardware DeviceType identities, carry endpoint include-tag intersections
   into scope-tag application, default new sources to endpoint include-scope,
   and fail legacy include-tag sources closed until an operator explicitly saves
   an endpoint-scope opt-out.
10. Exclude CIMC SNMP endpoints from standalone `dcim.device` emission while
    retaining the APIC CIMC inventory-item map as the canonical representation.
11. Treat dependency-preview workload estimates as apply work rather than exact
    drift, avoiding candidate/delete double counting and false drift claims.
12. Restore the smoke management command removed in 2.0 on the current
    single-branch backend. Select an existing configured source internally and
    keep credentials and private source identifiers out of shell arguments,
    output, and evidence artifacts.

## Decision Log

- Preserving NQE diff primary keys was rejected after live syntax probes showed
  that this Forward runtime rejects every tested combination of parameterized
  `@query` and `@primaryKey`. Removing the invalid annotation restores full
  execution; the normal fetcher fallback preserves diff/hybrid sync behavior.
- Counting endpoint-only scope as sufficient evidence for destructive prune was
  rejected; modeled-device scope remains the safety signal.
- Reusing the shared development database for release tests was rejected while
  unrelated active jobs are present; CI-style tasks must use their isolated
  fallback.
- Treating SoftwareVersion as a global Forward catalog was rejected because the
  DLM UI represents installed software. DeviceSoftware now creates and links a
  version in one operation; the catalog query can only enrich an existing row.
- Treating full SNMP `sysDescr` as DeviceType identity was rejected because
  console-server firmware and build metadata changes independently of hardware.
  Vendor overlays derive stable hardware models; generic endpoints use one
  manufacturer-scoped type.
- Preserving an implicit legacy endpoint-scope opt-out was rejected because it
  reimports untagged endpoint devices. A legacy source with include tags but no
  explicit marker now fails closed; saving the form records either the safe
  default or an intentional opt-out.

## Bundled changes

- **Endpoint-aware scope reconciliation** - imported SNMP endpoints are queried
  with the same scope rules used during sync and are no longer mislabeled as
  out-of-scope orphans.
- **Fail-closed endpoint protection** - reconciliation aborts when the endpoint
  scope probe fails instead of treating previously imported endpoint devices as
  removable.
- **Operator visibility** - the reconciliation UI and audit payload report the
  number of in-scope SNMP endpoints.
- **Endpoint scope tags** - endpoint include-tag intersections now feed the same
  NetBox device-tag application path as modeled devices.
- **Stable console-server identity** - Opengear and Avocent endpoints use the
  Console Server role and hardware-stable DeviceTypes instead of software-bearing
  `sysDescr` values.
- **Safe source defaults** - new sources default endpoint include-scope on;
  legacy include-tag sources fail closed until an operator explicitly saves an
  opt-out.
- **Drift report semantics** - workload estimates display as candidate rows,
  deletes, and apply work instead of being double-counted as exact drift.
- **CIMC identity** - CIMC endpoints are not imported as standalone devices;
  APIC CIMC inventory items remain authoritative.
- **Release-gate repair** - `invoke architecture-audit-check` now runs focused
  model, fetch, and query contract tests instead of calling a management command
  removed by the 2.0 single-branch teardown.
- **Bay-safe field validation** - `forward_smoke_sync` is restored for the
  current single-branch backend and can select an existing configured source
  without moving credentials or private identifiers through shell arguments or
  evidence.
- **DLM CVE/Vulnerability execution** - remove unsupported `@primaryKey`
  annotations that prevented both parameterized published queries from parsing;
  republish bundled queries after upgrading.
- **DLM installed-software integrity** - device-scoped software rows now carry
  lifecycle dates and create the SoftwareVersion plus DeviceSoftware link
  together; catalog rows cannot create zero-device versions.
- **Encrypted support-bundle dependency** - require `pyzipper` 0.4.0+ to remove
  the small-entry CRC32 disclosure reported as `PYSEC-2026-3044`.
- Python and NQE; no database migration.
