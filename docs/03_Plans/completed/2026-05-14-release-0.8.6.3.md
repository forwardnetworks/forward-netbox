# 0.8.6.3 Release

## Goal

Ship the routing, virtual-chassis, and fast-bootstrap hardening patch that
prevents optional beta routing rows and stale virtual-chassis output from
blocking otherwise usable baselines.

## Constraints

- Keep the release on the NetBox `4.5.x` line validated with NetBox `4.5.9`.
- Keep NQE as the source of row normalization.
- Do not introduce customer-specific IDs, snapshots, screenshots, or tenant
  data into tests or documentation.
- Keep optional routing behavior beta-scoped and row-tolerant.

## Touched Surfaces

- Package and plugin version metadata.
- Public release tables and install examples.
- Routing, peering, virtual-chassis, and fast-bootstrap hardening already
  captured in the routing/VC hardening plan.
- Forward org repository NQE copies for the updated routing maps.

## Approach

Publish `0.8.6.3` as a narrow patch release on top of `0.8.6.2`.

The release includes exact BGP scope lookup by router and VRF, NQE and
Python-side guards for invalid BGP ASN values, conservative skipping for
virtual-chassis rows that do not carry a real NetBox member position, and
fast-bootstrap baseline readiness when only optional beta model issues remain.

## Validation

- Focused tests for BGP scope lookup, ASN guard behavior, virtual-chassis
  no-op rows, fast-bootstrap optional issue behavior, and query registry
  assertions.
- Live Forward API execution of the updated routing and diagnostic NQE files
  with a small row limit.
- Forward org repository queries updated for the affected routing maps.
- Full release gate: `invoke ci`.
- GitHub CI on `main` and the release tag before public artifact publication.

## Rollback

Revert the release metadata/docs commit and keep `0.8.6.2` as current if
validation fails before publication. If publication succeeds and a
release-blocking issue is found, publish a newer patch release because PyPI
artifacts are immutable.

## Decision Log

- Chosen: patch release `0.8.6.3` because the change hardens existing beta and
  sync behavior without expanding the supported NetBox model surface.
- Rejected: delaying to a larger feature release because the fixes address
  current customer-visible sync failures.
