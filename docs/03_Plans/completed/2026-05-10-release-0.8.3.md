# Release Plan: 0.8.3

## Goal

Publish `0.8.3` for model failure isolation and virtual-chassis contract
hardening so one bad model row does not prevent later models, such as routing,
from running.

## Constraints

- Keep the release on the NetBox `4.5.x` line validated with NetBox `4.5.9`.
- Keep NQE as the source of normalized NetBox-ready data; do not invent missing
  virtual-chassis positions in Python.
- Do not mark an import with row issues as a clean diff baseline.
- Do not commit customer identifiers, network IDs, snapshot IDs, credentials,
  or screenshots.
- Require the full local release gate before tag and artifact publication.

## Touched Surfaces

- Package and plugin version metadata.
- Release compatibility tables and install examples.
- Query/planning failure isolation and multi-branch lifecycle behavior.
- Virtual-chassis row contract and adapter guard.
- Troubleshooting documentation and harness plan evidence.
- GitHub release and PyPI artifacts.

## Approach

1. Bump package metadata, plugin metadata, and Forward API user-agent to
   `0.8.3`.
2. Update public release tables and install examples.
3. Keep the previously validated failure-isolation implementation in the same
   release commit.
4. Re-run the full local release gate and artifact checks.
5. Commit, tag, push, publish GitHub release assets, and upload the same
   artifacts to PyPI.

## Validation

- `invoke ci`
- `python -m twine check dist/*`
- GitHub Actions success on `main` and the `v0.8.3` tag after push.
- GitHub release and PyPI project verification after publication.

## Rollback

If validation fails before publication, revert the `0.8.3` release commit and
keep `0.8.2` as current. If publication succeeds and a release-blocking issue is
found, publish a newer patch release because GitHub and PyPI artifacts are
immutable.

## Decision Log

- Chosen: `0.8.3` minor patch-level release because the change materially
  improves large-sync resilience without expanding the public model surface.
- Rejected: delaying to a larger feature release because stale query output can
  currently suppress unrelated routing work.
