# 0.8.6 Release

## Goal

Ship a point release that refreshes the Forward org-repository NQE path, tightens
IPAM prefix hygiene, and improves diagnostics for large customer baselines
without changing the NetBox-native sync workflow.

## Constraints

- Keep NQE as the source of NetBox-ready data shaping.
- Do not commit customer identifiers, network IDs, snapshot IDs, screenshots, or
  credentials.
- Keep the release compatible with the validated NetBox 4.5.x line.
- Preserve existing query-path and query-ID workflows for already-published maps.

## Touched Surfaces

- Built-in IPv4 prefix NQE.
- Query publishing helpers for flattened built-in NQE.
- Query-fetch diagnostics and row failure reporting.
- Device virtual chassis guard handling.
- User and reference documentation.
- Focused NetBox test coverage for the changed paths.

## Approach

- Publish built-in NQE with local imports flattened so org-repository query IDs
  do not depend on plugin-local import files.
- Filter IPv4 prefix host routes plus clearly non-importable any/loopback route
  artifacts in NQE instead of mutating prefix values in Python.
- Add a read-only full-baseline diagnostic for IP addresses that lack an
  imported covering prefix in the same VRF.
- Keep virtual chassis conservative by ignoring device rows that reference a
  virtual chassis without a member position.
- Serialize model objects before recording structured ingestion issue fields.

## Validation

- Live Forward API validation of the changed IPv4 prefix query against the
  selected source.
- Forward org repository publish and post-publish query-ID execution for the
  changed IPv4 prefix map.
- Focused NetBox tests for prefix query filters, query publishing, and
  parent-prefix diagnostics.
- Full release gate: `invoke ci`.

## Rollback

Revert the release commit and publish a follow-up Forward org repository commit
restoring the previous `forward_prefixes_ipv4` query source if needed.

## Decision Log

- Rejected Python-side prefix rewriting because it would invent address meaning
  after NQE and break the source-of-truth boundary.
- Rejected running parent-prefix diagnostics on diff workloads because diffs do
  not include the full prefix universe and would create false positives.
