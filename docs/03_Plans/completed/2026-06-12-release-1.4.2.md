# Forward NetBox 1.4.2 Release Cut

## Goal

Ship `forward-netbox` `1.4.2` as the current release line after the latest
production hardening pass, and publish the same validated tree to GitHub and
PyPI.

## Scope

- Keep the hard parent-device sync contract in place so child models cannot run
  without `dcim.device`.
- Keep query-ID drift remediation and the sync-detail repair entrypoints in
  place so local query updates stay aligned with the saved IDs.
- Keep support-bundle diagnostics and dependency preview available so sync
  failures can be traced with more complete evidence.
- Separate CIMC-managed hardware from APIC and ACI platform labels.
- Publish the validated package metadata, docs, and release surfaces as the
  `1.4.2` patch line.

## Constraints

- Preserve the validated 1.4 branch behavior and do not introduce new feature
  work during the release cut.
- Keep customer-identifying data out of tracked release text.
- Keep the GitHub release body consistent with the tagged commit and PyPI
  artifacts.

## Touched Surfaces

- `README.md`
- `docs/README.md`
- `docs/01_User_Guide/README.md`
- `forward_netbox/__init__.py`
- `pyproject.toml`
- `forward_netbox/queries/netbox_utilities.nqe`
- `forward_netbox/templates/forward_netbox/forwardsync.html`
- `forward_netbox/templates/forward_netbox/forwardsync_dependency_preview.html`
- `forward_netbox/tests/test_health.py`
- `forward_netbox/tests/test_query_registry.py`
- `forward_netbox/views.py`

## Approach

1. Validate the full local release harness.
2. Commit the release tree with release trailers.
3. Push the release commit to `main`.
4. Tag the commit as `v1.4.2`.
5. Publish the GitHub release from the tagged commit.
6. Upload the matching wheel and sdist to PyPI.

## Validation

- `rtk .venv/bin/invoke harness-check`
- `rtk .venv/bin/invoke ci`

## Rollback

- Delete the release tag and GitHub release if publication fails before the
  PyPI upload.
- Revert the release commit if the release cut itself is wrong.

## Decision Log

- Chosen: keep the release note focused on publication and validation rather
  than adding more product behavior during the cut.
- Chosen: describe the current release as `1.4.2` so the published package and
  the release notes match.

## Release Notes Draft

Forward NetBox `1.4.2` is a patch release that tightens operator visibility and
keeps the ACI/APIC/CIMC platform contract explicit.

Highlights:

- Separates CIMC-managed hardware into its own `CIMC` platform label while
  preserving the existing `APIC` and `ACI` behavior.
- Surfaces query-drift repair and dependency preview entrypoints on the sync
  detail page so operators do not need to jump to Health first.
- Keeps the module-bay merge hardening and parent-interface description
  preservation from the prior patch line.
- Retains the existing support-bundle diagnostics so release evidence and field
  troubleshooting stay actionable.
