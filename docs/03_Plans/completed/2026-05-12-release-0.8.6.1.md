# 0.8.6.1 Release

## Goal

Ship a narrow patch release that makes the NQE map bulk edit workflow clearer
for operators binding maps to Forward org-repository queries.

## Constraints

- Preserve the existing repository-path execution behavior.
- Do not add a separate bulk direct-query-ID workflow.
- Keep the workflow NetBox-native through the existing bulk edit form.

## Touched Surfaces

- NQE map bulk edit form labels and help text.
- Playwright UI harness expectations.
- Form tests.
- User-facing release and configuration documentation.

## Approach

- Rename the bulk operation labels to make repository-path mode explicit.
- Update field help text to state that bulk edit does not store direct query
  IDs and that query IDs are resolved at sync/diff time.
- Rename the per-map fieldset to `Map Query Path Choices`.
- Keep the underlying saved fields unchanged: `query_repository`, `query_path`,
  and optional `commit_id`.

## Validation

- `ForwardNQEMapBulkEditFormTest`
- `npm run test:ui`
- `invoke lint`
- Release gate: `invoke ci`

## Rollback

Revert this release commit. No data migration or Forward org repository change
is required.

## Decision Log

- Rejected direct query-ID bulk binding because repository paths are portable
  across orgs and already resolve to query IDs during sync and diff execution.
