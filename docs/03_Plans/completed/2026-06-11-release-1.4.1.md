# Forward NetBox 1.4.1 Release Cut

## Goal

Ship `forward-netbox` `1.4.1` as the current release line after the 1.4
hardening tranche, and publish the same validated tree to GitHub and PyPI.

## Scope

- Keep the hard parent-device sync contract in place so child models cannot run
  without `dcim.device`.
- Keep query-ID drift remediation in place so local query updates are detected
  when the saved query ID is still stale.
- Keep support-bundle diagnostics available so sync failures can be traced with
  more complete evidence.
- Publish the validated package metadata, docs, and release surfaces as the
  1.4.1 patch line.

## Constraints

- Preserve the 1.4 release line behavior; this is a release cut, not a feature
  expansion.
- Keep customer-identifying data out of tracked release text.
- Keep the GitHub release body consistent with the tagged commit and PyPI
  artifacts.

## Approach

1. Validate the full local release harness.
2. Commit the release tree with release trailers.
3. Push the release commit to `main`.
4. Tag the commit as `v1.4.1`.
5. Publish the GitHub release from the tagged commit.
6. Upload the matching wheel and sdist to PyPI.

## Validation

- `rtk .venv/bin/invoke harness-check`
- `rtk .venv/bin/invoke ci`
- `rtk .venv/bin/invoke build`

## Rollback

- Delete the release tag and GitHub release if publication fails before the
  PyPI upload.
- Revert the release commit if the release cut itself is wrong.

## Decision Log

- Chosen: use the repository release notes plan as the GitHub release body so
  the tagged commit and the published notes stay aligned.
- Chosen: keep the release cut focused on publication and validation rather
  than adding more product behavior during the cut.

## Release Notes Draft

Forward NetBox `1.4.1` is a patch release that hardens sync correctness and
release observability.

Highlights:

- Enforces the hard parent-device sync contract so dependent shards cannot run
  before `dcim.device` is available.
- Detects query-ID drift so the shipped NQE definitions and the saved query IDs
  stay aligned.
- Keeps support-bundle diagnostics attached to release readiness so field
  reports carry better evidence.
- Preserves the existing 1.4 hardening line while making the release tree
  suitable for GitHub and PyPI publication.
