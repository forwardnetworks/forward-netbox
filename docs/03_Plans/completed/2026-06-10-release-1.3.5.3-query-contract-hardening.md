# Forward NetBox 1.3.5.3 Query Contract Hardening

## Goal

Ship `forward-netbox` `1.3.5.3` with the last contract hardening pass from the
1.3.5.x line:

1. Keep the claimed-step and payload-compaction protections already proven in
   `1.3.5.2`.
2. Enforce strict shipped-query parameter contracts.
3. Strip legacy tag aliases from runtime NQE payloads.
4. Keep support-bundle previews summary-only.

## Scope

- Add runtime validation so unexpected NQE parameters fail fast instead of being
  injected silently.
- Keep legacy source-form aliases available for UI compatibility, but do not
  pass them through to NQE execution.
- Add regression coverage for shipped query parameter declarations and replay a
  mixed support-bundle contract shape.
- Bump release metadata and user-facing package documentation to `1.3.5.3`.

## Constraints

- Do not reintroduce column-filter shard fetches.
- Do not weaken the shipped query contract by allowing implicit parameters.
- Keep the release limited to contract hardening and release metadata.

## Approach

1. Validate the final runtime parameter payload against the declared query
   contract.
2. Remove legacy `device_tag_*` aliases from the runtime NQE payload.
3. Assert that every shipped builtin query seeds exactly the params it declares.
4. Replay a support-bundle case that includes both tag-capable and shard-only
   steps.
5. Update package metadata and release tables to the new patch version.

## Validation

- `rtk invoke check`
- `rtk invoke lint`
- Focused isolated tests for query contract compatibility and replay coverage

## Decision Log

- Kept source alias compatibility at the UI/form layer to avoid breaking saved
  sync configurations, while stripping the aliases before NQE execution.
- Chose hard fail validation for unexpected runtime params to keep the contract
  explicit and supportable.

## Rollback

- Remove `_validate_query_parameters` and the alias stripping in
  `query_fetch_execution.py`.
- Revert the added contract tests and release metadata version bump.
