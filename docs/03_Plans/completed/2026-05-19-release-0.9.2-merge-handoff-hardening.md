# Release Plan: 0.9.2

## Goal

Publish `0.9.2` with runtime hardening for resumable Branching shard execution,
specifically the merge handoff state where shard execution could stall on a
completed merge job marker.

## Constraints

- Keep NetBox-native Branching execution behavior.
- Preserve existing execution-ledger schema and API surfaces.
- Use additive hardening with idempotent reconciliation behavior.
- Include regression tests for state-machine transitions.

## Touched Surfaces

- `.github/workflows/ci.yml`
- `forward_netbox/jobs.py`
- `forward_netbox/utilities/multi_branch_executor.py`
- `forward_netbox/utilities/execution_ledger_reconciliation.py`
- `forward_netbox/utilities/execution_ledger_run_store.py`
- `forward_netbox/utilities/resumable_branching.py`
- `forward_netbox/utilities/sync_state.py`
- `forward_netbox/utilities/ingestion_merge.py`
- `forward_netbox/tests/test_jobs.py`
- `forward_netbox/tests/test_sync.py`
- `forward_netbox/tests/test_synthetic_scenarios.py`

## Approach

1. Stage execution hardening
   - Pass claimed shard index from job claim into executor.
   - Force executor to honor claimed index and fail fast when unresolved.

2. Reconciliation hardening
   - Normalize duplicate inflight stage steps to one canonical inflight step.
   - Clear stale pending step job bindings.

3. Merge handoff hardening
   - On duplicate merge worker entry (branch already merged/missing), mark
     execution step merged and enqueue next shard for auto-merge runs.
   - Emit explicit transition logs for merge handoff visibility.

4. Sync detail payload compaction
   - Keep the sync detail `Parameters` card responsive by showing summarized
     branch-run state (including `plan_items_count`) instead of rendering the
     full `plan_items` array.

5. CI harness dependency alignment
   - Install `invoke` and `python-dotenv` in GitHub Actions validation
     dependencies so `scripts/tests/test_tasks.py` imports match the runtime
     surface used locally.
## Validation

- Run targeted Django tests for stage claim, merge handoff, reconciliation, and
  sync-state behavior.
- Confirm live canary rollover beyond prior stuck shard.
- Confirm sync detail `Parameters` rendering remains responsive with large shard
  plans.

## Rollback

- Revert the release commit if local/remote CI fails before tag publication.
- If release is published and regression is found, ship forward patch release;
  avoid force-moving published tags.

## Decision Log

- Chosen: harden merge handoff in both worker and reconciliation paths to keep
  behavior resilient under duplicate jobs and stale state.
- Chosen: keep changes surgical in current architecture rather than reworking
  the run-state model during release stabilization.
- Chosen: compact branch-run display payload in sync Parameters while preserving
  full details in execution-run/log export surfaces.
- Rejected: release without explicit merge-handoff completion handling because
  canary showed real `merge_queued` stall behavior.
