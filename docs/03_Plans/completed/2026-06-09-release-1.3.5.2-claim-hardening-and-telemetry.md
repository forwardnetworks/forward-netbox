# Forward NetBox 1.3.5.2 Claimed-Step and Telemetry Payload Hardening

## Goal

Ship `forward-netbox` `1.3.5.2` to address two late-stage regressions reported from
large live synces:

1. Claimed execution steps drifting to stale persisted plan metadata during resume/overlap.
2. Sync detail, support bundle, and log payloads returning full `plan_items` arrays and overwhelming operators.

## Scope

- Keep execution scope attached to the in-memory claimed step while it is present.
- Preserve green-path execution behavior for normal (non-resumed) sync flows.
- Compact `plan_items` surfaces in UI and bundle payloads while retaining:
  - `plan_items_count` for exact telemetry,
  - the first previewed rows for operator diagnostics,
  - and exact behavior for tests that depend on count and truncation signals.
- Keep this patch limited to claimed-step resolution and payload shaping; no data-model rewrite.

## Constraints

- Do not change sync outcome semantics outside claimed-step path resolution and payload shaping.
- Do not alter default query behavior for unrelated models or non-claim execution paths.
- Preserve per-model query contracts and avoid changing persisted plan-item storage schema.

## Touched Surfaces

- `forward_netbox/jobs.py`
- `forward_netbox/utilities/multi_branch_executor.py`
- `forward_netbox/utilities/sync_state.py`
- `forward_netbox/views.py`
- `forward_netbox/tests/test_jobs.py`
- `forward_netbox/tests/test_sync.py`
- `forward_netbox/tests/test_sync_state.py`
- `forward_netbox/tests/test_log_export.py`

## Approach

1. Capture the claimed execution step in the job runner and pass it into
   `run_next_plan_item`.
2. Prioritize claimed-step metadata when loading shard scope in the execution engine.
3. Keep persisted index lookup as fallback only for paths without a claimed step.
4. Compact execution/preview payloads by adding count metadata and truncating
   detail arrays above a fixed `plan_items` limit.

- `rtk invoke test-isolated --test-label=forward_netbox.tests.test_sync.ForwardMultiBranchExecutorAdaptiveSplitTest`
- `rtk invoke test-isolated --test-label=forward_netbox.tests.test_log_export.ForwardIngestionLogExportViewTest`
- Focused regression tests for `plan_items` summary truncation and claimed-step scope selection.

## Decision Log

- Chose the claimed-step path over persisted index lookup to avoid stale scope
  execution when workers are resumed or overlapping.
- Added bounded payload summaries instead of removing telemetry entirely to preserve operator context.

## Rollback

- Revert claimed-step snapshot propagation in `stage_forward_branch_item` and
  `_run_plan_item`.
- Remove payload compaction helpers in `sync_state.py` and `views.py` and restore full
  `plan_items` inclusion.
