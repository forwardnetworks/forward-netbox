# Forward NetBox 1.3.5.4 Release Cut

## Goal

Ship `forward-netbox` `1.3.5.4` as the current patch release line for the
1.3.5.x branch. This release repackages the `1.3.5.3` query-contract hardening
on a fresh patch tag and updates the release metadata surfaces accordingly.

## Scope

- Bump package and plugin version metadata to `1.3.5.4`.
- Update the release tables and install examples in the repository docs.
- Validate the repository with the standard CI gate.
- Tag `v1.3.5.4`, publish the GitHub release, and upload the wheel/sdist to
  PyPI.

## Constraints

- Do not change the functional behavior already shipped in `1.3.5.3`.
- Keep the release notes honest that this is a patch release cut, not a new
  feature tranche.
- Keep customer-identifying content out of tracked files and release text.

## Touched Surfaces

- `pyproject.toml`
- `forward_netbox/__init__.py`
- `README.md`
- `docs/README.md`
- `docs/01_User_Guide/README.md`

## Approach

1. Bump the project and plugin versions to `1.3.5.4`.
2. Refresh the release compatibility tables and package install examples.
3. Run the full repository CI gate to verify the tagged tree.
4. Publish the release artifacts from the validated commit.

## Validation

- `rtk invoke ci`
- GitHub release page for `v1.3.5.4`
- PyPI project page for `forward-netbox 1.3.5.4`

## Decision Log

- Kept the functional release surface identical to `1.3.5.3` and used the new
  patch tag to make the current release state explicit.

## Rollback

- Retag the prior `v1.3.5.3` release as current only if the `1.3.5.4` publish
  is rejected.
- Revert the version and docs bump if the final validation gate fails.
