# Release Plan: 0.6.3

## Goal

Publish `0.6.3` with native NetBox LAG modeling for Forward aggregate interfaces, physical member `Interface.lag` assignment, and shard-safe placeholder handling during multi-branch imports.

## Constraints

- Keep committed artifacts free of customer identifiers, network IDs, snapshot IDs, credentials, and screenshots.
- Preserve Forward MTU values as reported by the NQE data model instead of guessing jumbo MTU locally.
- Keep the release aligned with the live Forward org query path used for validation.

## Touched Surfaces

- `forward_netbox/queries/forward_interfaces.nqe`
- `forward_netbox/utilities/sync.py`
- `forward_netbox/tests/test_query_registry.py`
- `forward_netbox/tests/test_sync.py`
- `README.md`
- `docs/README.md`
- `docs/01_User_Guide/README.md`
- `docs/02_Reference/built-in-nqe-maps.md`
- `docs/02_Reference/model-mapping-matrix.md`
- Package metadata and plugin version

## Approach

1. Bump package and plugin version metadata to `0.6.3`.
2. Update release compatibility tables and install examples.
3. Record the interface LAG implementation as the new release note.
4. Re-run the repo release gates before publish.

## Validation

- `python -m compileall forward_netbox/utilities/sync.py forward_netbox/tests/test_sync.py forward_netbox/tests/test_query_registry.py`
- Focused Docker Django tests for the interface query and LAG adapter behavior.
- Live Forward org query update and query-by-ID smoke against the updated interface map.
- `invoke ci`

## Rollback

Revert the version bumps and release-note edits, then restore `0.6.2` release metadata if the LAG behavior needs to be backed out.

## Decision Log

- Chosen: ship as `0.6.3` because this is a patch-level import behavior correction and does not expand the public workflow surface.
- Rejected: deferring until a major release because the native LAG model is the correct NetBox shape for the current Forward interface data.
