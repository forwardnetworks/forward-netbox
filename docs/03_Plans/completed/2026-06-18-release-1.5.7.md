# Release 1.5.7

## Goal

Ship the fixes accumulated since 1.5.6, the most urgent being the **504 gateway
timeout** Partner hit clicking **Prune orphans** on a large (5157-device) fabric:
the synchronous prune cascade-deleted ~200 devices (interfaces, IPs, change-log
signals) and exceeded the proxy's 30s timeout, rolling the transaction back so
nothing was pruned.

## Constraints

- Heavy mutating UI actions must not run synchronously in the request on large
  fabrics.
- Keep the read-only previews (Scope Reconciliation, Module Readiness) — they
  load fast enough — but move the writes to background jobs.
- No customer data, credentials, or network IDs in repo/tests.

## Bundled changes

1. **Prune orphans + Create missing module bays → background jobs.** The POST
   views enqueue `forward_netbox.jobs.prune_forward_orphans` /
   `create_forward_module_bays` and redirect immediately ("Queued job #N; watch
   the Jobs tab"). No request-time timeout.
2. **Module readiness `Ready` reflects bays only.** `Ready` was `No` even after
   all bays existed, because module rows for out-of-scope devices
   (`missing_device_rows`) counted against it. Those skip harmlessly, so `Ready`
   now keys on `missing_bay_rows == 0`.
3. **Bulk ipaddress duplicate-global IP** (already on main from the prior commit):
   the bulk update path used `clean_fields()` instead of `full_clean()` so a
   pre-existing duplicate global IP no longer fails the row.

## Approach

Move the two heavy mutating UI actions to NetBox background jobs: the POST views
call `Job.enqueue(...)` against new `forward_netbox.jobs` runners and redirect
immediately; the runners call the existing `prune_orphan_devices` /
`create_missing_module_bays` utilities and terminate the job with the result.
The read-only previews stay synchronous. Module readiness `ready` drops the
`missing_device_rows == 0` clause. Then bump the version, run the full CI mirror,
and release through the standard branch → CI → main → tag → GitHub → PyPI flow.

## Touched Surfaces

- `forward_netbox/jobs.py` — `prune_forward_orphans`, `create_forward_module_bays`.
- `forward_netbox/views.py` — the two POST views enqueue jobs.
- `forward_netbox/utilities/module_readiness.py` — `ready` semantics.
- `forward_netbox/utilities/apply_engine_bulk.py` — bulk dup fix (prior commit).
- Tests: `test_scope_module_ui` (job enqueue + runner), `test_module_readiness`,
  `test_ipaddress_dup_global`.
- `docs/01_User_Guide/troubleshooting.md`; version bump + 3 release tables.

## Validation

- `forward_netbox.tests.test_scope_module_ui`, `test_module_readiness`,
  `test_ipaddress_dup_global`, `test_device_scope_reconciliation_audit_command`.
- Full suite; local CI mirror (pre-commit clean + run twice; harness; harness
  tests; py_compile; mkdocs --strict; build). GitHub CI both matrices.

## Rollback

Revert the job runners + view enqueues (restore synchronous POST), the `ready`
clause, and the version bump. No schema/data impact.

## Decision Log

- Jobs over raising the proxy timeout: the timeout is operator infra; the plugin
  fix is to not hold the request. Mirrors the existing sync/validation jobs.
- Read-only previews stay synchronous (they load within the timeout); only the
  writes (prune, create bays) became jobs.
- Preview Dependencies (a pre-existing synchronous view) also times out on this
  fabric — noted as a separate follow-up, not bundled here.
