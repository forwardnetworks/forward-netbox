# Release 2.1.3 — prune empty orphan sites

**Date:** 2026-06-28

## Goal
Cut 2.1.3 bundling the empty-orphan-site prune feature (see
`2026-06-28-prune-empty-orphan-sites.md`).

## Constraints
- Drop-in from 2.1.2; no schema/data change. Prune semantics extended: the
  existing "Prune orphans" action now also deletes sites that are absent from
  the Forward location result AND have zero devices and zero racks.

## Touched Surfaces
- Version bump: `pyproject.toml`, `forward_netbox/__init__.py`, the 3 README
  tables, `CHANGELOG.md`.
- Feature: empty-orphan-site prune — see `2026-06-28-prune-empty-orphan-sites.md`.

## Approach
Standard release: prepare bumps version + README + CHANGELOG; verify (lint,
harness, sensitive, build); publish FF main + tag + GitHub release + PyPI.

## Validation
Full suite (903). Lint/harness/sensitive. PR CI green before merge; main CI green
after. No `.nqe` change this release, so no CustomerOrg org query republish needed.

## Rollback
Revert the release commit; `pip install forward-netbox==2.1.2`.

## Decision Log
- Bundled the feature with the version bump in one release branch rather than a
  separate docs PR, matching the 2.1.1/2.1.2 single-branch release pattern.
