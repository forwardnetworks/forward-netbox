# Release 2.0.3 — module-sync warning cap, CI changelog gate fix, doc + dead-code hygiene

**Date:** 2026-06-25

## Goal
Operator-facing + repo-health patch, drop-in from 2.0.2, no engine/schema/org
changes: (1) stop module-sync readiness warnings from flooding the log, (2) fix
the `CHANGELOG matches README` CI gate that has been red on every release since
v1.7.2, (3) remove stale internal docs and now-dead executor code.

## Problem
- **Module-sync warning spam.** When module sync is enabled before a device's
  module bays exist, EVERY module row on that hardware skips with `module bay
  <bay> does not exist on <device>. Run forward_module_readiness ...`. One
  systemic readiness condition logged up to `CONFLICT_WARNING_DETAIL_LIMIT` (20)
  near-identical lines per sync.
- **Red CI on main.** `scripts/gen_changelog.py --check` stamped each version
  header with its `git tag` date. The release flow commits CHANGELOG.md *before*
  the tag exists, so the committed file is dateless; CI runs after the tag is
  pushed and regenerates a dated header → permanent mismatch. The `CI` workflow
  (separate from the PR `Validate` job, push-only) has been red since v1.7.2.
- **Stale docs + dead code.** `technical-debt.md` described removed modules
  (`multi_branch_planner.py`, `multi_branch_executor.py`, `multi_branch.py`) and
  shard/resume vocabulary — it misled a code audit. `ForwardFastBootstrapExecutor.run()`
  and its `_record_change_totals` / `_raise_if_blocking_issues_exist` helpers are
  dead: the class is never instantiated in prod and `ForwardSingleBranchExecutor`
  overrides `run()`.

## Constraints
- Drop-in from 2.0.2: no engine, schema, migration, or Forward org changes.
- Idempotent and non-regressing: skip reasons other than `missing-module-bay`
  keep the default detail cap (byte-identical output); the retained executor base
  helpers (`_create_ingestion`, `_set_runtime_phase`, `_query_preflight_enabled`,
  `_sync_mode`) must stay intact for `ForwardSingleBranchExecutor`.
- `gen_changelog.py` output stays a pure function of README.md (deterministic
  across commit-time, CI-time, and tag-push timing).
- Only unreachable code/tests removed — verified by call-site, not by name.

## Approach
- Add a per-reason detail cap `SKIP_WARNING_DETAIL_LIMITS = {"missing-module-bay":
  3}`; the aggregate-skip reporter uses it (helper `_skip_warning_detail_limit`).
  Reasons not listed keep the default 20, so `missing-interface` is byte-identical.
- Make `gen_changelog.py` a pure function of README.md by dropping the per-version
  tag-date suffix (dates live in git tags / GitHub releases). `--check` is now
  deterministic regardless of tag-push timing.
- Rewrite the stale `technical-debt.md` sections to the 2.0 single-branch reality.
- Delete `ForwardFastBootstrapExecutor.run()` + the two run-only helpers + dead
  imports, and the dead `ForwardFastBootstrapExecutorTest` class (7 `test_run_*`
  methods). Keep the class + its live base helpers (`_create_ingestion`,
  `_set_runtime_phase`, `_query_preflight_enabled`, `_sync_mode`) that the
  single-branch executor reuses. Class name kept to avoid a cross-module rename.

## Touched Surfaces
- `forward_netbox/utilities/sync.py` — `SKIP_WARNING_DETAIL_LIMITS`.
- `forward_netbox/utilities/sync_reporting.py` — `_skip_warning_detail_limit`; per-reason cap in record/emit.
- `forward_netbox/utilities/fast_bootstrap_executor.py` — drop dead `run()` + 2 helpers + dead imports.
- `scripts/gen_changelog.py` — drop tag-date dependency; `CHANGELOG.md` regenerated dateless.
- `docs/03_Plans/technical-debt.md` — refresh stale branch-execution / harness-layer sections.
- Tests: `test_sync.py` — add module-bay cap test; remove the dead FB executor test class + its now-unused imports.

## Validation
Unit: `test_apply_dcim_module_caps_missing_module_bay_warnings` (3 + summary) +
`test_apply_dcim_macaddress_aggregates_missing_interface_warnings` unchanged (21).
`gen_changelog.py --check` exits 0 deterministically. Full suite green
(`single_branch_executor` coverage exercises the retained base helpers).
Lint/harness/sensitive. **GitHub `CI` workflow green** (the previously-red gate).

## Rollback
Additive reporter change + a docs edit + deletion of unreachable code/tests +
a deterministic-changelog generator. Revert restores 2.0.2.

## Decision Log
- Per-reason cap (3) over globally lowering `CONFLICT_WARNING_DETAIL_LIMIT`: the
  default 20 is right for sporadic per-row anomalies; only the systemic readiness
  reason needs fewer, so scope it and leave all others byte-identical.
- Dropped CHANGELOG dates rather than reorder the release to tag-before-commit:
  removing the non-deterministic input is the smallest fix that makes the gate
  green permanently and for historical commits' regeneration; dates remain in
  tags + GitHub releases.
- Included the `fast_bootstrap.run()` teardown in this patch (not a later minor):
  it is pure dead-code/test removal with no behavior change, and bundling it keeps
  the cleanup in one reviewed step. The class name rename is deferred separately.
