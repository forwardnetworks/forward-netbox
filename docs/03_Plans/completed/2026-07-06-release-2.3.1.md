# Release 2.3.1 — tag idempotency + drift-report staleness

**Date:** 2026-07-06

## Goal
Two fixes reported by the design partner after upgrading to 2.3.0:
1. A sync produced an ingestion error `create extras.tag: {'name': ['Tag with
   this Name already exists.']}` — a tag whose NAME exists but whose SLUG did
   not match was re-created instead of reused.
2. The drift report kept showing large drift after a successful sync because it
   is rendered from the last dependency-preview payload, which predated the
   sync — no signal that the numbers were stale.

## Constraints
- Behaviour-compatible patch (2.3.1); no schema/migration change.
- Reuse existing infrastructure: the tag cache already indexes by slug AND name;
  `_ensure_scope_tag` already coalesces on `(slug,), (name,)`.

## Touched Surfaces
- `forward_netbox/utilities/sync_interface.py` — `apply_extras_taggeditem`
  coalesce_sets `[("slug",)]` → `[("slug",), ("name",)]` (reuse an existing
  same-named tag instead of re-creating it).
- `forward_netbox/views.py` — `ForwardSyncDriftReportView` flags the report
  stale when the sync's last ingestion is newer than the preview job.
- `forward_netbox/templates/forward_netbox/forwardsync_drift_report.html` —
  staleness warning banner.
- Tests: `test_sync.py` (name/slug reuse regression), `test_health.py`
  (drift stale / not-stale view tests).
- Version bump: `pyproject.toml`, `forward_netbox/__init__.py`, README compat
  tables, `CHANGELOG.md`.

## Approach
The tag fix mirrors the proven `_ensure_scope_tag` coalesce order (slug first,
name fallback); the unique-lookup cache and `_prime_tag_cache` already support
the name key. The drift staleness compares `sync.last_ingestion.created` against
the preview job's `created`.

## Validation
`docker … manage.py test forward_netbox.tests.test_sync
forward_netbox.tests.test_health` (322 green) + the 3 new tests; full suite;
harness / sensitive / pre-commit / mkdocs.

## Rollback
`pip install forward-netbox==2.3.0`. Code-only; no migration.

## Decision Log
- Match by name as a FALLBACK (not primary): slug stays the identity for
  Forward-managed tags; name only rescues a pre-existing divergent-slug tag.
- Drift staleness is a display flag, not an auto-refresh: re-running the
  preview is the operator's explicit action; auto-running a heavy dry-run on
  every page view would be surprising.

## Bundled changes
2.3.1 is a patch on 2.3.0.

Fixes
- Feature/scope tags are now matched by name as well as slug when applying, so
  an existing tag with the same name but a different slug is reused instead of
  raising "Tag with this Name already exists".
- The drift report now warns when it is older than the latest sync (it is
  computed from the cached dependency-preview, so a sync run afterwards leaves
  stale "everything to create" numbers) — re-run Preview Dependencies for a
  current view.
