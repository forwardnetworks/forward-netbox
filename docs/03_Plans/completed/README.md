# Completed Plans

This directory preserves closed implementation history. A plan is closed when
its behavior shipped, its decision was explicitly rejected, or a newer
architecture/release plan superseded it. Historical status and deferral wording
describes the decision at that time and does not override current production
code, `ARCHITECTURE.md`, the technical-debt tracker, or the active release plan.

The 2.6 handoff moved every pre-2.6 plan out of `active/`. There is no implied
claim that every exploratory proposal shipped; the claim is that none remains an
unowned active obligation.
