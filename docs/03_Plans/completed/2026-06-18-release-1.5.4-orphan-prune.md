# Release 1.5.4 — Orphan Device Prune

## Goal

Give operators a safe way to delete the stale "orphan" devices a tag-scoped sync
leaves in NetBox. Diagnosed live on the ORG/REDACTED org: the Forward `N.Patel`
scope resolves to ~2204 completed devices (matches the expected ~2196), but
NetBox held 2418 — ~214 orphans from an earlier, broader sync.
`device_tag_prune_out_of_scope` cannot remove them: it only deletes rows the sync
query *returns* that fall outside scope, and orphans are absent from the scoped
result entirely.

## Constraints

- Operator-invoked and reviewable, not an automatic sync-time delete of unmarked
  devices (dangerous for mixed-source NetBox instances).
- Dry-run by default; deletion only with an explicit `--apply`.
- Never delete tagged-but-backfilled devices (real, just not collected this
  snapshot) — only devices not tagged in the Forward result at all.
- No customer data, credentials, or network IDs in repo/tests.

## Touched Surfaces

- `forward_netbox/management/commands/forward_device_scope_reconciliation_audit.py`
  — `--prune-orphans` (+ `--apply`) deletes the `out_of_scope` set in batches
  inside a transaction; corrected the remediation text (prune does NOT reach
  orphans).
- `forward_netbox/tests/test_device_scope_reconciliation_audit_command.py` —
  dry-run-keeps and apply-deletes-only-out-of-scope tests.
- `docs/01_User_Guide/troubleshooting.md` — corrected cause + `--prune-orphans`
  remediation.
- `pyproject.toml`, `forward_netbox/__init__.py`, the three release tables — bump
  to 1.5.4.

## Approach

The reconciliation audit already computes `out_of_scope = netbox_names -
tagged_names`. `--prune-orphans` reports that set as the prune candidate; with
`--apply` it deletes those devices (batched `Device.objects.filter(name__in=...)`
in a transaction). Cascade removes each device's interfaces/IPs, which is the
intended effect of removing the device.

## Validation

- `forward_netbox.tests.test_device_scope_reconciliation_audit_command`
  (dry-run keeps devices; apply deletes only the untagged out-of-scope device,
  preserving in-scope and backfilled).
- Full `forward_netbox.tests` suite; local CI mirror (pre-commit, harness +
  harness tests, py_compile, mkdocs --strict, Django check, build).
- GitHub CI green on both NetBox matrices before merge/tag/publish.

## Rollback

Revert the command flags + tests and the version bump. The prior release (1.5.3)
remains published. No schema/data/migration impact.

## Decision Log

- Operator-run prune over automatic sync-time delete: auto-deleting unmarked
  NetBox devices is unsafe for instances not exclusively managed by one sync, so
  the safe primitive is a reviewable command with dry-run default.
- Prune set is `netbox - tagged` (not `netbox - completed`) so tagged-but-
  backfilled devices are preserved — they are real devices, just stale in this
  snapshot.
- A future automatic, marker-based reconcile-delete (tag synced devices, then
  delete tagged devices that leave scope) can build on `apply_device_scope_tags`;
  it would not catch pre-existing unmarked orphans, which is exactly what this
  command handles.
