# Release Plan: 0.8.4

## Goal

Publish `0.8.4` for virtual chassis HA classification, Forward repository
query lookup compatibility, and clearer sync failure progress visibility.

## Constraints

- Keep Forward NQE as the source of truth for model classification.
- Keep NetBox-native `dcim.virtualchassis` support available for custom maps
  that emit true virtual chassis rows.
- Do not include customer identifiers, network IDs, snapshot IDs, credentials,
  or screenshots in tracked files.
- Publish GitHub and PyPI artifacts only after local and GitHub validation pass.

## Touched Surfaces

- Version metadata in `forward_netbox/__init__.py` and `pyproject.toml`
- Built-in virtual chassis NQE
- Forward API committed-query lookup helper
- Sync run-state progress and failure activity helpers
- Query registry, sync, and Forward API tests
- User and reference documentation

## Approach

Release the already-tested virtual chassis behavior change together with a
progress-state correction exposed by field evidence. The bundled virtual
chassis query emits no rows by default because Forward HA peer data is not the
same as NetBox virtual chassis membership. Python guards remain in place for
stale repository queries or custom virtual chassis maps.

Clear row-level heartbeat fields when the executor enters a new phase or marks
the sync failed, so the Forward Sync detail page reports the terminal failure
instead of the last successful row heartbeat.

## Validation

Completed:

- Forward Org Repository update for `/forward_netbox_validation/forward_virtual_chassis`
  was published through the plugin's native query binding helper.
- The committed repository query source hash matched the local bundled query.
- Executing the committed query against the resolved current snapshot returned
  zero rows.
- Added regression coverage for clearing stale row progress on phase changes
  and failed syncs.
- `invoke sensitive-check`
- `invoke ci`
- `python -m twine check dist/*`

## Rollback

If publication fails, delete the local tag before pushing or delete the remote
tag/release before PyPI upload. If the release is already on PyPI, publish a
follow-up patch restoring the prior virtual chassis query and Forward API
lookup behavior.

## Decision Log

- Chosen: minor patch release `0.8.4` because this now includes an operator
  visibility correction in addition to the virtual-chassis query and lookup
  compatibility fixes.
- Rejected: mapping Forward vPC/MLAG/cluster HA into NetBox virtual chassis.
  Those are HA/control-plane peer relationships, not shared-control-plane
  chassis membership.
