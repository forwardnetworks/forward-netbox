# Release Plan: 0.5.2.1 Patch

## Goal

Publish `0.5.2.1` containing the already-implemented patch fixes for plugin version display and inferred-cable query parser compatibility.

## Constraints

- Keep behavior changes narrowly scoped to release/version metadata and already-validated cable query compatibility fix.
- Preserve NetBox/Branching native workflows and existing query-id execution behavior.
- Pass project CI gates before publishing artifacts.

## Touched Surfaces

- `pyproject.toml`
- `forward_netbox/__init__.py`
- `README.md`
- `docs/README.md`

## Approach

1. Bump package and plugin versions to `0.5.2.1`.
2. Update release documentation tables and install example references.
3. Run `pre-commit run --all-files` and `invoke ci`.
4. Tag and publish GitHub release assets.
5. Upload wheel/sdist to PyPI.

## Rollback

- If publishing fails after tag creation, issue a superseding patch release.
- If PyPI upload fails partially, do not overwrite artifacts; rebuild and publish a superseding patch.

## Validation

- `pre-commit run --all-files`
- `invoke ci`
- verify GitHub release creation
- verify PyPI artifact upload

## Decision Log

- Keep release patch scope minimal; no additional functional changes beyond already-implemented fixes.
- Require full CI gate prior to tagging/publishing.
