# Release 2.0.5 — "Forward Field Integration" rebrand + brand polish

**Date:** 2026-06-25

## Goal
Present the plugin as **Forward Field Integration** and add a light Forward
Networks visual identity (logo + accent), with zero functional change.

## Constraints
- Display-only. No change to the Python package `forward_netbox`, the PyPI
  distribution `forward-netbox`, `base_url="forward"`, NQE query names, models,
  APIs, or the `"Publish Forward NetBox NQE maps"` commit contract.
- Theme-safe: must look correct in NetBox light AND dark themes.
- No global NetBox style overrides — brand styling is self-contained to the
  plugin's own pages.

## Touched Surfaces
- `forward_netbox/__init__.py` — `verbose_name` → "Forward Field Integration".
- `forward_netbox/navigation.py` — sidebar `PluginMenu` label → "Forward Field Integration".
- `README.md`, `docs/README.md`, `mkdocs.yml` — product/site titles (and dropped a stale "fast bootstrap backend" line).
- `forward_netbox/static/forward_netbox/img/fn-logo.svg` — Forward wordmark, wordmark recolored to `currentColor` (theme-aware), `#ff3506` accent kept.
- `forward_netbox/templates/forward_netbox/inc/brand_bar.html` — self-contained brand bar (logo + "Field Integration" + tagline, `#ff3506` left accent).
- `forwardsource.html` / `forwardsync.html` / `forwardingestion.html` — include the brand bar at the top of `{% block content %}`.

## Approach
The logo's wordmark was `fill="#fff"` (invisible on NetBox's light theme); changed
to `currentColor` so it inherits page text color and reads in both themes, while
the `#ff3506` accent stays constant. The brand bar is a self-contained include
(scoped `.fwd-brand-bar` styles only) so it can never clash with NetBox's own
light/dark chrome. The logo ships as a standard plugin static asset (served via
`collectstatic` → nginx/whitenoise).

## Validation
Plugin boots with the new `verbose_name`/menu label; `base_url`/`name` unchanged.
Source, Sync, and Ingestion detail pages render 200 with the brand bar present;
logo collects to the static root. No test asserts the old brand strings. Full
suite green; lint/harness/sensitive. Brand bar previewed in both themes before
release.

## Rollback
Display-only and additive. Reverting the touched files (and removing the static
asset + include) restores 2.0.4; no schema/data/API change.

## Decision Log
- Kept the full name "Forward Field Integration" consistently (NetBox nav wraps
  fine) rather than abbreviating in the sidebar.
- Brand bar scoped to the plugin's 3 primary detail pages (Source/Sync/
  Ingestion) via a shared include — not a global NetBox `head_extra` restyle —
  to keep blast radius zero and avoid theme clashes.
- Did NOT rename the package/dist/URL/query-names/commit-contract; renaming those
  breaks installs, bookmarks, the org NQE library, and tests for no user benefit.
