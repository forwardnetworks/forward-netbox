# Release 1.5.8

## Goal

Fix two issues REDACTED hit on 1.5.6 once module sync was enabled on the large
fabric:

1. **`dcim.module` IntegrityError** — `duplicate key value violates unique
   constraint "dcim_interface_unique_device_name" ... GigabitEthernet0/0/0
   already exists` (24 such errors). Creating a module replicates the module
   type's component templates, whose names collide with the device interfaces
   Forward already syncs independently.
2. **`ipam.fhrpgroup` churn** — every full sync deletes and recreates the same
   ~13 HSRP groups (Created 13 / Deleted 13, identical names). Recurring, not a
   one-time re-keying.
3. **Preview Dependencies 504** — the `forwardsync_dependency_preview` GET ran a
   heavy live multi-branch dry-run synchronously, exceeding the proxy's ~30s
   timeout on the large fabric (same 504 class as the 1.5.7 prune/create-bays
   actions).

## Constraints

- Fix at the plugin layer; no change to the published org NQE (no ORG org push).
- Keep module-only components (no existing match) still created.
- Must not over-suppress genuine deletes.

## Touched Surfaces

- `forward_netbox/utilities/sync_primitives.py` — `coalesce_update_or_create` /
  `coalesce_upsert` / `upsert_values_from_defaults` accept optional
  `create_instance_attrs`.
- `forward_netbox/utilities/sync_runner_adapters.py` — `_upsert_values_from_defaults`
  forwards it.
- `forward_netbox/utilities/sync_inventory_module.py` — module apply passes
  `{"_adopt_components": True}`.
- `forward_netbox/utilities/sync_runner_contracts.py` — `_split_diff_rows` drops
  delete rows whose coalesce identity is also being upserted in the same batch.
- `forward_netbox/jobs.py` — `forward_dependency_preview` job runner.
- `forward_netbox/views.py` — `ForwardSyncDependencyPreviewView` GET renders the
  latest completed preview job's cached payload; new POST enqueues the job.
- `forward_netbox/templates/forward_netbox/forwardsync.html` — Preview
  Dependencies becomes a POST button plus a View Last Preview link.
- Tests: `test_sync` (module adopt), `test_sync_runner_contracts` (diff dedup),
  `test_health` (dependency preview job + cached render).
- Version bump + 3 release tables; troubleshooting note.

## Approach

**Module IntegrityError:** NetBox `Module.save()` honours `_adopt_components` —
when set, an installed component (module__isnull) whose name matches a module
type template is attached to the module (UPDATE) instead of recreated (INSERT).
Thread an optional `create_instance_attrs` dict through the upsert helpers so the
module apply sets `_adopt_components=True` on the new `Module` before save. Forward
interfaces get adopted into the module; module-only templates are still created.

**FHRP churn:** The FHRP NQE emits one row per `(device, interface, state)` with
`select distinct row`; HSRP state flapping active<->standby between snapshots makes
the snapshot diff emit an ADDED variant and a DELETED variant for the SAME group
identity `(protocol, group_id, address, vrf)`. `_split_diff_rows` routed ADDED to
upsert and DELETED to delete, so the group was recreated and deleted every run.
Add a final reconcile: drop any delete row whose `_coalesce_identity` matches an
upsert row's identity in the same batch. Model-agnostic — covers any model where
several rows collapse onto one NetBox identity.

**Preview Dependencies 504:** mirror the 1.5.7 prune/create-bays pattern. A new
`forward_dependency_preview` job runs `_dependency_dry_run_payload(sync)` and
caches the JSON on `job.data`. The Preview Dependencies button becomes a POST
that enqueues the job and redirects; the preview GET page renders the latest
completed job's cached payload (and serves `?format=json` from it), so it never
calls Forward in-request and cannot 504. A View Last Preview link opens the
cached page.

## Validation

- `forward_netbox.tests.test_sync.ForwardSyncRunnerTest.test_apply_dcim_module_adopts_existing_interface`
  and the existing module apply tests.
- `forward_netbox.tests.test_sync_runner_contracts` (drop-delete + keep-distinct).
- Full suite; local CI mirror (pre-commit clean + twice; harness; harness tests;
  py_compile; mkdocs --strict; build). GitHub CI both matrices.

## Rollback

Revert the `create_instance_attrs` threading + the module flag, and the
`_split_diff_rows` reconcile block, and the version bump. No schema/data impact.

## Decision Log

- Adopt over disable-replication: adoption attaches existing Forward interfaces to
  the module (correct ownership) while still creating module-only components;
  disabling replication would leave module-only components missing.
- FHRP fixed at the diff layer, not the NQE: the apply-layer reconcile is general
  (any many-rows-per-identity model) and needs no org NQE republish.
- Reconcile only drops a delete when the SAME identity is upserted in the same
  batch, so genuine deletes (distinct identity) are preserved.
