# Release Plan: 0.8.2

## Goal

Publish `0.8.2` for the NQE map query-reference workflow and IP address skip handling updates.

## Constraints

- Keep the workflow NetBox-native: list view selection plus the built-in `Edit Selected` bulk action.
- Do not commit customer identifiers, network IDs, snapshot IDs, screenshots, or credentials.
- Preserve NQE as the normalization/source-of-truth layer; NetBox stores native map references and executes the selected query mode.
- Do not publish release artifacts until local validation passes.

## Touched Surfaces

- `forward_netbox/models.py` and migration metadata for query reference fields.
- `forward_netbox/forms.py`, `views.py`, `api/`, tables, filters, and templates for query selectors and bulk edit.
- `forward_netbox/utilities/forward_api.py`, `query_fetch.py`, `query_registry.py`, and `query_binding.py`.
- Built-in NQE, generated reference docs, user docs, and UI harness coverage.
- Package version metadata and release compatibility tables.

## Approach

1. Add repository query path as a first-class NQE map execution mode.
2. Use Forward-backed selectors for query path, direct query ID, folder, and commit lookup.
3. Keep bulk operations inside NetBox's native bulk edit surface and make the selected-map binding explicit.
4. Add the reverse operation in the same bulk edit form so selected maps can be restored to bundled raw query text.
5. Validate with focused tests, full harness checks, Playwright UI, docs, package build, and sensitive-content scanning.

## Decision Log

- Chosen: repository `query_path` as the preferred portable map reference because direct query IDs are org-specific.
- Chosen: per-selected-map query selectors in native bulk edit because map names are not a reliable binding contract.
- Chosen: restore only when the bundled query can be identified unambiguously; custom or ambiguous maps are skipped instead of guessed.
- Rejected: a custom bulk binding page because it would duplicate NetBox's native bulk edit workflow.

## Scope

- Portable repository `query_path` execution for Forward NQE maps.
- NetBox-native NQE map selectors for repository paths, direct query IDs, and commit IDs.
- Native bulk edit support for publishing bundled NQE into the Forward Org Repository, binding selected maps to repository paths, and restoring selected maps to bundled raw NQE.
- NQE/IP address handling for rows whose Forward interface cannot be resolved.
- Release metadata, package version, docs, migration, and validation evidence.

## Validation

- Focused NQE map form and binding tests passed before the release bump.
- Full release validation must pass before tag and artifact publication:
  - `invoke harness-check`
  - `invoke harness-test`
  - `invoke lint`
  - `invoke check`
  - `invoke scenario-test`
  - `invoke test`
  - `npm run test:ui`
  - `invoke docs`
  - `python -m build`
  - `python -m twine check dist/*`
  - sensitive-content scan and `git diff --check`

## Rollback

If validation fails before publication, revert the `0.8.2` release commit and keep `0.8.1.1` as current. If publication succeeds and a release-blocking issue is found, publish a newer patch release rather than replacing immutable artifacts.
