# Release 2.3.0 — GA / enterprise hardening

**Date:** 2026-07-04

## Goal
Cut 2.3.0 as the enterprise/GA-readiness release: it bundles the post-2.2.5
hardening (security, supply-chain, observability, reliability, dead-code removal)
and moves the project framing to a supported product. Drop-in from 2.2.5 on the
same NetBox 4.6.4 baseline.

## Constraints
- No NetBox/branching baseline change (still 4.6.4 / 1.1.0+).
- Backwards compatible: stored credentials auto-encrypt on save; old rows migrate.
- Prominent upgrade caveats in the release notes (see Bundled changes).

## Touched Surfaces
Version bump only in this plan (`pyproject.toml`, `forward_netbox/__init__.py`, the
three README compatibility tables, `CHANGELOG.md`). The feature/fix surfaces are
each covered by their own plan docs in `docs/03_Plans/active/2026-07-0{3,4}-*.md`
(hygiene, multi_branch removal, density-learning removal, enterprise tranches 1/2,
SaaS clamp, retry jitter + PK prune, credential encryption, identity-aware scope,
metrics, stuck-job alert, CI coverage/constraints, GA framing).

## Approach
`invoke release 2.3.0 --write` (bump + tables + changelog + local CI mirror), then
the full Django suite, then commit. Publish is deferred to a later turn
(`--publish`/`--finish` via the tag-triggered Trusted-Publishing workflow).

## Validation
Full Django suite on 4.6.4 (green); lint / harness / sensitive / mkdocs --strict;
`makemigrations --check` clean (0031 is a data migration); coverage floor 70%
(baseline 76%).

## Rollback
`pip install forward-netbox==2.2.5`. Migration 0031 reverses (decrypts). All other
changes are code/docs/CI only.

## Decision Log
- Minor (2.3.0) not major: additive + backwards compatible; only operational
  caveats (SECRET_KEY rotation) apply, no breaking API/schema change.
- Ship the whole hardening tranche as one release rather than a string of patches:
  it is a coherent GA-readiness step.

## Bundled changes
2.3.0 is an enterprise/GA-readiness release. Drop-in from 2.2.5 on NetBox 4.6.4.

Security
- **The Forward API password is now encrypted at rest** (Fernet, keyed off Django
  `SECRET_KEY`). Existing passwords are migrated automatically on upgrade.
  **Caveat:** rotating `SECRET_KEY` makes stored passwords undecryptable — re-enter
  the password on each Forward source after any `SECRET_KEY` rotation.
- Vulnerability-disclosure policy (`SECURITY.md`), Dependabot, `pip-audit` against a
  pinned `constraints.txt`, and a `makemigrations --check` drift guard in CI.
- The doc site no longer publishes internal/field planning material; a committed
  customer name was scrubbed and the content scanner hardened to block recurrences
  via a CI secret.
- Sync attribution no longer silently runs as an arbitrary superuser; the missing
  invoking-user fallback is logged.

Supply chain / release
- **PyPI Trusted Publishing** (OIDC) with PEP 740 attestations, plus a per-release
  CycloneDX **SBOM** — the manual laptop token is retired.

Observability
- **Prometheus metrics** exporter (`forward_metrics`) and a **stuck-job alert**
  command (`forward_stuck_job_alert`).

Reliability / data safety
- API retry backoff now has jitter and honors `Retry-After` (capped); SaaS request
  rate is clamped to a safe ceiling on save; device prune deletes by resolved PK
  (identity-tracked) instead of the non-unique device name.

Quality / maintenance
- Removed the dead `multi_branch` scaffolding and the dead density-learning write
  loop; added an automated populated-DB upgrade test across the destructive
  migrations; CI enforces a 70% coverage floor (baseline 76%).

Docs / framing
- Now presented as an officially maintained, supported Forward Networks
  integration; adds UPGRADE/ROLLBACK, versioning/deprecation, and REST API
  reference docs.

Known limitations (unchanged this release)
- Scope membership is name-keyed: a Forward-scoped device name conservatively
  protects every same-named NetBox device across sites (full cross-site
  disambiguation needs Forward location mapping).
- The optional `netbox-routing` / `netbox-peering-manager` integrations are not
  loadable on NetBox 4.6 at their current pinned versions, so those tests skip.
- Sync writes intentionally bypass NetBox object-level permissions — restrict who
  can create or trigger a sync (see `SECURITY.md`).
