# 2026-05-20 Release 0.9.4.3 Protected Delete Skip

## Goal
Ship a tiny patch release that hardens delete behavior when NetBox protected foreign-key references block object deletion.

## Constraints
- Keep behavior NetBox-native; no forced cascading deletes across plugin boundaries.
- Keep scope narrow to delete error handling and regression tests.
- Exclude runtime evidence artifacts from release commits.

## Touched Surfaces
- `forward_netbox/utilities/sync_primitives.py`
- `forward_netbox/tests/test_sync.py`
- `pyproject.toml`
- `forward_netbox/__init__.py`

## Approach
1. Catch Django `ProtectedError` in coalesce delete path.
2. Convert it to `ForwardDependencySkipError` with row context.
3. Add regression test proving protected delete references map to dependency skip.
4. Bump package and plugin version to `0.9.4.3`.

## Validation
- `invoke harness-check`
- `invoke test`
- `invoke lint`
- `invoke check`
- `invoke docs`
- `invoke ci`

## Rollback
Revert the delete handling and version bump commits, then rerun validation gates.

## Decision Log
- Chose skip-on-protected-reference over forced dependency deletion to preserve model ownership and avoid destructive side effects.
