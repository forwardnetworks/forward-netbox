# Release 1.2.2

## Goal

Publish `forward-netbox` `1.2.2` to GitHub Releases and PyPI.

## Constraints

- Release only intended repo changes.
- Keep customer identifiers, credentials, network IDs, snapshot IDs, and screenshots out of tracked content.
- Preserve the `1.2.x` compatibility line for NetBox `4.5.x` and `4.6.x`.
- Use the repo virtual environment on this Mac for `python`, `pre-commit`, build, and publish tooling.

## Touched Surfaces

- Package version metadata.
- Release compatibility and version-history docs.
- Git commit, tag, GitHub Release, and PyPI artifacts.

## Approach

1. Bump package and plugin metadata to `1.2.2`.
2. Update public release notes and install examples.
3. Run local release gates and package checks.
4. Commit, tag, push, and verify GitHub CI.
5. Build artifacts from the tagged commit.
6. Publish GitHub Release and PyPI package.
7. Verify GitHub and PyPI publish state.
