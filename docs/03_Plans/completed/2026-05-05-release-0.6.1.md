# Release Plan: 0.6.1

## Goal

Publish `0.6.1` with the NetBox-unassignable interface IP filter and diagnostics, while keeping the native module import surface marked beta.

## Constraints

- Keep committed artifacts free of customer identifiers, network IDs, snapshot IDs, credentials, and screenshots.
- Preserve NetBox-native behavior by filtering rows NetBox cannot assign instead of mutating Forward-reported addresses.
- Update Forward Org Repository query IDs used by the live validation org so query-ID syncs receive the same NQE correction.
- Release to GitHub Releases and PyPI only after local CI and sensitive-content checks pass.

## Touched Surfaces

- `forward_netbox/queries/forward_ip_addresses.nqe`
- `forward_netbox/queries/forward_ip_addresses_unassignable_diagnostics.nqe`
- `forward_netbox/utilities/query_fetch.py`
- `forward_netbox/utilities/query_registry.py`
- `forward_netbox/utilities/sync.py`
- Package metadata, README files, reference docs, and release notes

## Approach

1. Bump package and plugin metadata to `0.6.1`.
2. Update compatibility tables and version history with the IP address filter and diagnostic behavior.
3. Commit the updated `Forward IP Addresses` NQE source to the Forward Org Repository path used by the deployed query-ID map.
4. Run the full repository CI gate, build artifacts, and publish the GitHub and PyPI release.
5. Verify GitHub Actions after pushing the release commit and tag.

## Validation

- `invoke ci`
- `python -m build`
- `python -m twine check dist/forward_netbox-0.6.1*`
- Forward Org Repository dry-run commit and post-commit query-ID validation
- GitHub Actions checks for the pushed release commit and tag

## Rollback

Publish a follow-up patch release that reverts the IP address NQE filter, diagnostic query, adapter guard, and metadata bump. Forward Org Repository query changes can be reverted with a new org repository commit restoring the previous query source.

## Decision Log

- Chosen: `0.6.1` patch release because the change fixes invalid IP rows without expanding the public model surface beyond the `0.6.0` beta module work.
- Rejected: delaying for another minor release because the customer-facing failure is isolated to IP address ingestion and query-ID users need the corrected NQE promptly.
