# Release Plan: 0.8.1.1

## Goal

Publish `0.8.1.1` as a narrow patch release for the virtual chassis NQE position fix.

## Constraints

- Keep the release on the NetBox `4.5.x` line.
- Do not include customer identifiers, network IDs, snapshot IDs, credentials, or private screenshots.
- Keep the fix in NQE so NetBox receives native `dcim.virtualchassis` fields.
- Publish immutable GitHub and PyPI artifacts only after local validation and GitHub Actions pass.

## Touched Surfaces

- Package and plugin version metadata
- Release compatibility and version-history docs
- Built-in virtual chassis NQE map and query registry tests
- GitHub Release and PyPI artifacts

## Approach

1. Preserve the already-validated `vc_position` NQE change.
2. Update release metadata and public docs to `0.8.1.1`.
3. Run the repo release gate and package checks.
4. Commit, tag, push, verify GitHub Actions on `main` and the tag.
5. Publish the GitHub Release and PyPI package.

## Validation

- `invoke ci`
- `python -m twine check dist/*`
- GitHub Actions success on `main` and `v0.8.1.1`
- GitHub Release and PyPI project page verified after publish

## Rollback

If validation fails before publishing, revert the `0.8.1.1` metadata/docs commit and keep `0.8.1` as the latest published release. If publication succeeds but a release-blocking issue is found, publish a newer patch release rather than replacing immutable artifacts.

## Decision Log

- Chosen: `0.8.1.1` patch release because the change is a narrow NQE correction on top of `0.8.1`.
- Rejected: delaying until the next feature release because query-ID users need the corrected virtual chassis field contract promptly.
