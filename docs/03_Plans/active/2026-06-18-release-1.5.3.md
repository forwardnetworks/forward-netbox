# Release 1.5.3

## Goal

Cut the 1.5.3 release: bump the plugin version, refresh the release-compatibility
tables, and ship the bulk-ORM hardening + REDACTED-driven fixes accumulated on
`feat/latest-collected-catchup`.

## Constraints

- Version bumped in lockstep across `pyproject.toml`, `forward_netbox/__init__.py`,
  and the three release-compatibility tables (root README, docs/README,
  docs/01_User_Guide/README) plus the install examples.
- All local and GitHub CI checks must pass before merge/tag/publish.
- NQE changes (APIC platform, IP address split) require re-publishing the bundled
  queries to the Forward org repositories (the CI validation-org step auto-runs
  `publish_builtin_nqe_map_queries`).
- No customer data, credentials, or network IDs in repo/tests.

## Touched Surfaces

- `pyproject.toml`, `forward_netbox/__init__.py` — version 1.5.2 -> 1.5.3.
- `README.md`, `docs/README.md`, `docs/01_User_Guide/README.md` — compatibility
  tables (new 1.5.3 row, 1.5.2 superseded) and install version pins.

## Approach

Bump version and tables, then run the full CI mirror locally (sensitive-content
all-history, pre-commit all-files, harness + harness tests, py_compile, mkdocs
build --strict, migrations, Django checks, synthetic + full plugin tests, build).
Push the branch to run GitHub CI on both NetBox 4.5.9 and 4.6.2 matrices, confirm
green, then merge to main, tag, GitHub release, and PyPI publish.

## Validation

- Local: every CI step above green.
- GitHub Actions `CI` workflow green on both NetBox matrix versions.
- `python -m build` produces the 1.5.3 wheel + sdist.

## Rollback

Revert the version bump commit; the prior release (1.5.2) remains the published
artifact. No schema/data impact from the bump itself; feature migrations
(0023) are independently reversible.

## Decision Log

- Cut 1.5.3 from `feat/latest-collected-catchup` as a single release bundling the
  bulk-ORM hardening, APIC platform fix, IP address map split, and the opt-in
  scope-tag + audit tooling — all parity-tested and churn-free.
- ipam.prefix remains adapter-only (bulk promotion deferred; blockers documented
  in the prefix-churn-investigation plan).
