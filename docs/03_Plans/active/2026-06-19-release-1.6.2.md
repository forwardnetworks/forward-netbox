# Release 1.6.2

## Goal

Ship the complete 1.6.x follow-up set under a clean version. 1.6.1 was published
to PyPI from a build that predated the FHRP churn fix and the device-panel
deep-link (both were added after 1.6.1 went out). PyPI versions are immutable, so
1.6.1 is yanked and 1.6.2 carries everything.

## Constraints

- PyPI 1.6.1 cannot be re-published; 1.6.2 must contain the full intended set.
- Yank 1.6.1 so nobody installs the partial build.
- No customer data/credentials/network ids in repo.

## Bundled changes (cumulative over 1.6.0)

- Device tag scope covers VLANs/VRFs (were unscoped); prefixes derive from
  connected interface subnets (not routing tables).
- FHRP group churn fixed via identity-bucket sharding (active/standby co-located
  so the diff dedup fires).
- Device analysis as a first-class NetBoxModel: fleet list view, REST API,
  filterset, nav, device-FK panel, opt-in post-sync refresh, Open in Forward
  deep-link.
- Collection-gap alert command, run-history drill-down, hardened release tooling,
  CHANGELOG generator + commitlint, harness pre-push simulation.

## Touched Surfaces

- Version bump (pyproject, `forward_netbox/__init__.py`) + 3 README tables +
  CHANGELOG. All feature code already landed in prior commits on this line.

## Approach

Bump to 1.6.2, full local CI mirror + Django suite, branch -> GitHub CI (both
matrices) -> fast-forward main -> tag v1.6.2 -> GitHub release -> PyPI. Restore
tag v1.6.1 to the actually-published commit and yank PyPI 1.6.1. Republish the
org NQE maps to CustomerOrg (VLAN/VRF/prefix query signatures changed).

## Validation

- Full `forward_netbox.tests` (1176, green on the underlying commit).
- Local CI mirror; harness `--base origin/main`; GitHub CI both matrices.
- Live CustomerOrg validation already done for the scoping/prefix queries.

## Rollback

Revert the version bump; the feature code stands independently. No schema/data
impact beyond migration 0024/0025 already shipped on the line.

## Decision Log

- 1.6.2 over re-publishing 1.6.1: PyPI immutability makes re-publishing 1.6.1
  impossible; a clean new version is the only way every user gets the FHRP fix +
  deep-link. 1.6.1 is yanked to prevent installs of the partial build.
