# Release 1.6.0

## Goal

Ship the blue-sky roadmap tranche as a minor release: operator-experience and
observability features plus release tooling, on top of the completed bulk-ORM
coverage from 1.5.10. Large enough to warrant a minor bump.

## Constraints

- GA Forward only (no Predict). Device analysis surfaces NQE-derivable proxies.
- New read-only features must not run live Forward calls on page render or block
  the request (jobs + cached payloads).
- Live-validate the new device-analysis NQE against real Forward data.
- No customer data, credentials, or network ids in repo/tests/docs.

## Bundled changes

1. **Release automation** — `scripts/release.py` + `invoke release` encode the
   full release flow with the CI gotchas baked in.
2. **Operations Guide + plan hygiene** — consolidated operator doc; shipped
   release plans archived to `completed/`.
3. **Collection-gap health signal** — backfilled-tag count surfaced in the sync
   health summary (cheap DB read).
4. **Sync observability — run history** — per-sync ingestion run history panel.
5. **Device analysis panel** — GA per-device reachability proxy, connectivity-
   degree blast radius, CVE exposure; NQE + model (migration 0024) + refresh job
   + device-detail panel. Live-validated against CustomerOrg (5284 devices).
6. **Bidirectional drift report** — per-model NetBox-vs-Forward drift derived from
   the cached dependency-preview payload.

## Touched Surfaces

- `scripts/release.py`, `scripts/tests/test_release.py`, `tasks.py`.
- `docs/01_User_Guide/operations.md`, `docs/03_Plans/{active,completed}/`.
- `forward_netbox/utilities/{health,run_history,device_analysis,drift_report,scope_reconciliation}.py`.
- `forward_netbox/{models,views,jobs,template_content}.py`, migration 0024.
- `forward_netbox/queries/forward_device_analysis.nqe`.
- Templates: run history, drift report, device analysis panel, health card,
  sync detail buttons.
- Tests: `test_health`, `test_scope_module_ui`, `scripts/tests/test_release`.
- Version bump to 1.6.0 + 3 compatibility tables.

## Approach

Each feature is independent and read-only over stored data or background jobs;
none change the Branching sync path. Cut as a normal release: bump + tables +
this plan, full local CI mirror, branch → GitHub CI (both matrices) → fast-forward
main → tag v1.6.0 → GitHub release → PyPI → sync local main.

## Validation

- Full `forward_netbox.tests` suite (Django) green.
- Local CI mirror: pre-commit clean + run twice, harness check, harness tests,
  py_compile, mkdocs --strict, build.
- NQE linter clean on the device-analysis query.
- **Live CustomerOrg validation**: device-analysis NQE returns expected fields/values
  over the real fabric (5284 devices) — confirms the bare-expression fix.
- GitHub CI green on NetBox 4.5.9 and 4.6.2.

## Rollback

Each feature reverts independently (see its own plan). The release itself rolls
back by reverting the version bump and table rows; migration 0024 adds an
isolated read-only table with no dependents.

## Decision Log

- Minor bump (1.6.0) not patch: the tranche adds new models, NQE, panels, and
  tooling — feature-level scope.
- Device-analysis blast radius is the GA connectivity-degree proxy; true path
  blast radius needs Forward path/Predict APIs (not device NQE), documented in
  the query and panel.
