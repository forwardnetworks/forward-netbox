# Release 2.5.9 - DLM observability and query-path publishing

## Goal

Ship the post-2.5.8 DLM diagnostics and eliminate the stale-query failure mode
caused by publishing built-in maps as fixed direct query IDs. Publishing from a
sync becomes the single operator action: update the Forward Org Repository and
bind matching enabled maps to live repository paths.

## Constraints

- Keep direct query IDs available for intentional custom maps.
- Preserve an existing `commit_id`; a blank commit follows repository head.
- Do not store customer identifiers, credentials, query IDs, network IDs, or
  snapshot IDs in release evidence.
- No migration and no bundled NQE source change.
- Publishing requires Forward NQE-library write permission.

## Touched Surfaces

- `forward_netbox/utilities/query_binding_resolution.py` owns publish writeback.
- `forward_netbox/views.py` and sync templates own the operator action.
- Query-drift and validation guidance point to Publish only.
- Query-binding, query-registry, Health, endpoint, and DLM tests preserve the
  behavior.
- `tasks.py` and the current single-branch integration tests restore a scenario
  gate that executes real tests instead of the retired empty 1.x module.
- Version metadata, compatibility tables, changelog, and operator docs describe
  the 2.5.9 upgrade action.

## Approach

1. Persist `query_repository` plus `query_path` after Publish and clear
   `query_id` and raw `query`.
2. Preserve an existing explicit `commit_id`; otherwise leave the path on head.
3. Remove the standalone Refresh Query IDs view and buttons.
4. Verify a path-bound map resolves the current query ID at execution time and
   follows a later repository-head update without another NetBox write.
5. Publish and verify the bundled query set in the configured ADP org without
   recording sensitive identifiers.
6. Repoint scenario and ingestion-delete gates at the current single-branch,
   bulk-merge, and stuck-run recovery architecture.

## Rollback

Install `forward-netbox==2.5.8`; no migration rollback is required. Org query
commits remain auditable in Forward. To restore fixed execution intentionally,
set a direct query ID or set `commit_id` on the repository-path map. Reverting
the NetBox package does not delete or rewrite published org queries.

## Decision Log

- Publish is the single repair action; a separate Refresh action was rejected
  because it recreated fixed IDs and allowed silent drift.
- Automatic refresh during every sync was rejected because it mutates query
  identity mid-workflow, adds a live repository dependency, and defeats pins.
- Repository paths remain the default portable identity; `commit_id` is the
  explicit reproducibility mechanism.
- Patch version 2.5.9 follows the project's existing feature-in-patch release
  convention and carries no schema migration.

## Bundled changes

- **Live query-path publishing** - Publish Bundled Queries updates the Forward
  Org Repository, clears stale direct query IDs, and binds matching enabled maps
  to repository paths that resolve current head at each sync. Existing explicit
  commit pins are preserved.
- **Simpler query repair workflow** - the standalone Refresh Query IDs action is
  removed from the sync detail and Health pages; query-drift and validation
  guidance now point to Publish only.
- **DLM readiness diagnostics** - Health reports enabled optional maps whose
  models are not selected, base/alias hardware-notice map mismatches, and
  missing DLM dependency readiness before a sync silently omits data.
- **DLM skip visibility** - dependency-related skips are summarized into
  ingestion issues with bounded examples and a rollup, and skip messages carry
  actionable model context.
- **Release-gate integrity** - scenario and ingestion-delete tasks no longer
  target a retired module that discovered zero tests; they exercise current
  single-branch ingest/delete, bulk merge, planning, and recovery behavior.
- Upgrade action: run **Publish Bundled Queries** once for each org-backed sync
  to publish the current bundled source and convert matching built-in maps to
  repository-path bindings.
- Python-only; no database migration and no bundled NQE source change.
