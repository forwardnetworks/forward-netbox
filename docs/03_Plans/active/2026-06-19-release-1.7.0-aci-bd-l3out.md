# Release 1.7.0 — ACI Bridge Domain and L3Out Graduation

## Goal

Graduate ACI bridge domain and L3Out NQE maps from `where false` stubs to
moquery-based command-inventory queries. Harden the query publish path and
platform adapter to eliminate no-op edits and manufacturer drift on update.

## Constraints

- Do not commit customer identifiers, network IDs, snapshot IDs, credentials,
  or raw live rows.
- Bridge domain and L3Out queries must use the moquery command-inventory pattern
  already established by VRF and tenant maps; no per-value NQE fetches.
- Platform adapter must preserve existing manufacturer on UPDATE for
  ACI/APIC/CIMC platforms to prevent operator-set overrides being wiped.
- Query publish must skip no-op edits (source unchanged) to reduce Forward API
  churn on `forward_validation_org_query_audit --repair`.
- `--overwrite` flag on the audit command must require explicit opt-in; default
  behavior (`--repair` without `--overwrite`) must remain additive-only.

## Bundled changes

1. **ACI bridge domain and L3Out NQE maps** — graduated from `where false` stubs
   to moquery-based command-inventory queries with full field extraction.
2. **ACI VRF and tenant moquery alignment** — switched to the same command-
   inventory pattern for consistency; no functional change to output rows.
3. **Query publish skip-no-op** — `publish_builtin_nqe_map_queries` now compares
   committed source before calling `edit_org_nqe_query`, eliminating redundant
   API writes on `--repair` runs.
4. **Platform manifest split (create vs update)** — `_ensure_platform` now uses
   separate `create_values` and `update_values` so manufacturer is set on CREATE
   but never overwritten on UPDATE.
5. **Validation audit `--overwrite` flag** — explicit opt-in to republish queries
   that differ from the bundled source (default `--repair` remains additive-only).

## Approach

1. Promote BD and L3Out NQE maps from `where false` stubs to moquery-pattern
   queries that read `forward_aci_command_inventory` and parse field values.
2. Align VRF and tenant maps to the same pattern for consistency.
3. Add `require_source_code=True` to live drift check so stale-source errors
   surface early.
4. Skip `client.edit_org_nqe_query` when committed source equals new source
   (pure no-op).
5. Split `_ensure_platform` `create_values` and `update_values` so manufacturer
   is only set on CREATE; UPDATE never touches manufacturer (preserves operator
   overrides).
6. Add `--overwrite` flag to `forward_validation_org_query_audit` for explicit
   republish of differing queries.

## Decision Log

- **BD/L3Out moquery pattern** — consistent with VRF/tenant maps; avoids
  per-value NQE fetches that would require sharding.
- **Skip no-op publish** — reduces Forward API write volume during audit repair
  runs; correctness unchanged since we only skip when source is identical.
- **`update_values` excludes manufacturer** — operators may set a custom
  manufacturer in NetBox; sync should not overwrite that on every run.
