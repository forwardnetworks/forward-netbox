# Release 2.0.2 ‚Äî scope-tagging for multi-tag "any" + VLAN-1 warning noise

**Date:** 2026-06-25

## Goal
Two operator-facing fixes, drop-in from 2.0.1, no org republish: (1) make
`apply_device_scope_tags` work under multiple include tags in `any` match mode
(tag each device with exactly the include tags it carries), and (2) silence the
spurious `Skipping untagged VLAN 1` warning.

## Constraints
- No engine, schema, or migration changes; no ¬ß3.2 org republish (org-published
  device queries untouched).
- Idempotent: re-sync produces zero tag/VLAN churn.
- Zero blast radius on the non-scoped device apply fast path.

## Touched Surfaces
- `forward_netbox/utilities/sync_interface.py` ‚Äî `_interface_untagged_vlan` skips VID 1 silently.
- `forward_netbox/utilities/query_fetch_execution.py` ‚Äî inline scope resolver emits `tagNames`, intersects per device, persists/loads `scoped_matched_tags`; `ForwardQueryContext` field + `as_dict`.
- `forward_netbox/utilities/multi_branch_lifecycle.py` ‚Äî thread `scoped_matched_tags` onto the branch runner (the make-or-break wire).
- `forward_netbox/utilities/sync.py` ‚Äî runner `_scope_matched_tags` default.
- `forward_netbox/utilities/sync_device.py` ‚Äî `_scope_tags_enabled`/`_ensure_scope_tag`/`_scope_managed_tags`; per-device add + bounded stale removal.
- `forward_netbox/utilities/apply_engine_bulk.py` ‚Äî bulk gate uses `_scope_tags_enabled`.
- Tests: `test_device_scope_tagging.py` (rewritten), `test_scope_matched_tags.py` (new), VLAN-1 behaviour.

## Approach
Design Approach B (no republish): the inline scope-resolution query already
iterates exactly the in-scope devices, so it additionally `select`s
`device.tagNames`, intersects with the sync's include tags at resolve time (small
payload), persists the per-device map in the existing context artifact, and the
executor threads it onto the runner. The device apply then tags each device with
its matched include tags and removes only this sync's stale scope tags (bounded
to the include-tag universe ‚Äî never user/feature tags). Single-tag/`all` is
byte-identical to before (intersection == all include tags). VLAN 1: the apply
skips it silently, matching `forward_vlans.nqe` which never imports VID 1.

## Validation
Unit: scope-tag apply (per-device any, all, idempotent, stale-removal preserving
user tags, disabled, unmatched) + resolver intersection + `as_dict` map. Full
suite 889. Live (clean DB, multi-tag scope DC_Hyd+DC_Lon, apply_device_scope_tags
on): each device tagged with exactly its tag (Hyd‚Üí30, Lon‚Üí25, **0 double-tagged**,
no skip warning, 0 tracebacks); re-sync **0 merge churn**. GitHub CI green.

## Rollback
Additive/defensive. Reverting the touched files restores 2.0.1; no schema/org
state changed.

## Decision Log
- Chose Approach B over A (edits the two hottest org-published device queries +
  republish + degrade window) and C (live NQE in the ~5000x apply loop) ‚Äî B has
  the smallest blast radius and reuses already-computed/persisted data.
- Stale-tag removal is bounded to this sync's include-tag NetBox Tags so it never
  strips user/feature tags; add/remove are no-ops at steady state ‚Üí 0 churn.
