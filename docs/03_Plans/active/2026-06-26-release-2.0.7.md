# Release 2.0.7 — MAC missing-interface consistency, IP-diagnostic noise, require_diff block clarity

**Date:** 2026-06-26

## Goal
Three operator-facing fixes from Blake's full-DC sync support bundle, drop-in from
2.0.6: stop a benign MAC condition showing as a hard failure, collapse two benign
IP-diagnostic walls, and make a `require_diff` validation block self-diagnosing.

## Constraints
- No engine/schema change; behavior parity with the IP path for the MAC fix.
- Logging/diagnostics only for the IP-diagnostic collapse and the block message.

## Touched Surfaces
- `forward_netbox/utilities/apply_engine_bulk.py` — bulk MAC interface lookup now falls back to `_lookup_interface` (canonical match, like the bulk IP path); a missing interface records an aggregated `missing-interface` skip instead of a hard `ForwardSearchError` "failed" issue.
- `forward_netbox/utilities/query_diagnostics.py` — "filtered unassignable" and "no imported parent prefix" diagnostics each emit ONE summary line (count + plain-English why + ≤3 examples + `(+N more)`) instead of a 20-row wall + suppression line.
- `forward_netbox/utilities/validation.py` — when the device-coverage/foundational gate blocks AND `diff_fallback_mode == require_diff`, append a remediation reason naming the cause (failed diff fetch under require_diff) and the `Allow full fallback` remedy + Refresh Query IDs.
- `forward_netbox/utilities/model_validation.py` — `clean_forward_sync` now REJECTS the `require_diff` + source `device_tag_prune_out_of_scope` combination at save/validate time (they are mutually exclusive — prune needs full execution, require_diff forbids it). This is the actual root cause behind Blake's blocked sync (`query_fetch_execution.py:1848` raises per model when both are set → all 19 models fail → device-coverage block). Rejecting at config time stops the cryptic 11-minute failure.
- Tests: `test_apply_engine.py` (MAC skip not failure), `test_models.py` (require_diff hint + require_diff/prune rejection).

## Approach
The bulk MAC path was the only interface-attach path that raised a hard error for a
missing interface; the IP and non-bulk-MAC paths treat it as a benign aggregated
skip. Mirror the bulk IP path exactly (lookup fallback + aggregated skip). The two
IP diagnostics already produced a good summary line; drop the per-row example loop.
The require_diff block was cryptic ("dcim.device query failed … depend on device
coverage") and hid that a diff fetch failed under require_diff — add a hint that
names the real cause and the one-setting recovery.

## Validation
Unit: MAC missing-interface → aggregated skip, no issue/failed-stat; require_diff
gate appends the remedy (allow_fallback gate unchanged). Full suite. Lint/harness/
sensitive. (The diff-fetch ROOT cause that triggered Blake's block needs the live
ingestion exception — not in the bundle — and is tracked separately.)

## Rollback
Localized to the three functions; reverting restores 2.0.6 behavior. No schema/data
change.

## Decision Log
- MAC fix mirrors the bulk IP path rather than inventing new behavior — guarantees
  parity and removes the misleading red "failed" issue for a benign skip.
- Kept the `missing-interface` skip visible (not rolled up) — it can flag a real
  apply gap; only demoted it from hard-failure to skip.
- Block-message hint only appended under `require_diff` (so existing allow_fallback
  blocks read identically) — pinned by test.
