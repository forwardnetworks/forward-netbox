# Release 2.1.0 — read-only stale global-IPAM audit

**Date:** 2026-06-26

## Goal
Device-tag scope prune is device-derived and never removes network-global IPAM
(prefixes, VLANs, VRFs), so NetBox can accumulate global objects Forward no longer
reports. Give operators a safe way to find them: a read-only audit that lists
NetBox global IPAM absent from the latest Forward fetch as review candidates. It
never deletes (auto-deleting shared global IPAM is high blast radius — see the
scope-prune coverage section of the Operations Guide).

## Constraints
- Read-only. The command and util never create, update, or delete anything.
- No false delete candidates: identity matching must equal what the sync uses, and
  any object with an indeterminate identity is reported `unmatchable`, never stale.
- No change to the apply path or to prune behavior.

## Touched Surfaces
- `forward_netbox/utilities/scope_ipam_audit.py` (new) — `audit_global_ipam_scope`
  / `audit_model_rows`. Reuses the apply engine's `lookup_key_from_object` /
  `lookup_key_from_values` and a read-only mirror of apply's FK resolution
  (vrf-by-name, vlan site-by-slug/name; prefix CIDR handled by the key helper).
- `forward_netbox/management/commands/forward_scope_ipam_audit.py` (new) — CLI:
  `--sync-name/--sync-id`, `--models`, `--limit`, `--fail-on-stale`. JSON output.
- `forward_netbox/tests/test_scope_ipam_audit.py` (new) — VRF/VLAN/prefix stale
  detection, CIDR normalization, vrf-scoped match, site-less VLAN unmatchable,
  enabled-model filtering with injected rows.
- `docs/01_User_Guide/operations.md` — "Auditing stale global IPAM" subsection.

## Approach
Fetch the sync's Forward rows for each enabled global IPAM model (read-only, via
`ForwardQueryFetcher`), build the Forward identity set with the exact apply key
helpers, then walk every NetBox object of that model and report those whose
identity matches no Forward row. The `fetch_rows` callable is injectable so tests
run without a live Forward client. Identity reuse (not re-derivation) is what makes
a "stale" verdict trustworthy enough for a human to act on.

## Validation
Unit: 5 audit tests (above). Command loads + `--help`. Full suite. Lint/harness/
sensitive. Not live-validated against a customer instance (read-only; safe).

## Rollback
Additive — two new files + a doc subsection. Remove them to revert; no behavior or
schema change elsewhere.

## Decision Log
- Read-only audit, not auto-prune: global IPAM is not device-owned, so "out of
  scope" is ill-defined and deleting shared objects (a /16 aggregate, a network
  VLAN) is high blast radius. The operator decides; the tool only surfaces.
- Reuse `lookup_key_from_object/values` rather than re-deriving identity, so the
  report matches apply semantics (no false candidates from FK/CIDR mismatches).
- `unmatchable` objects (no determinate key) are never flagged — a conservative
  bias so the audit cannot produce a wrong delete candidate.
- CLI-only (no UI delete button) keeps the surface small and safe for v1.
