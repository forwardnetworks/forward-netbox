# Release 2.6.0 - Scope And Convergence Safety

## Goal

Ship a general 2.6 workflow that resolves the remaining post-2.5.11 scope and
drift ambiguity: plugin-managed scope tags self-heal after every successful
sync, and operators can prove convergence from persisted same-snapshot sync
evidence without mistaking dependency workload for object-level drift.

## Constraints

- Require and test NetBox 4.6.5 with exactly netbox-branching 1.1.1.
- Preserve unrelated operator and feature tags on every device.
- Remove include tags only when `apply_device_scope_tags` is enabled, because
  that setting establishes plugin ownership of those NetBox assignments.
- Keep dependency-preview workload estimates labeled as upper bounds; do not
  present fetched rows as exact drift.
- Use persisted ingestion counters and snapshot IDs only. Report rendering must
  not make a live Forward request.
- Do not automatically delete devices or global catalog records.
- Do not commit customer names, tag values, device identities, support-bundle
  rows, network IDs, snapshot IDs, or screenshots.
- No bundled NQE change and no schema migration. Do not publish or tag the
  release until acceptance is separately authorized.

## Touched Surfaces

- Runtime/package/testbed compatibility metadata and startup diagnostics

- Harness knowledge freshness and scheduled gardening signal
- Focused scope, job, health, runtime dependency, and export tests
- Upgrade, operations, configuration, troubleshooting, and release docs

## Approach

1. Pin package resolution, CI, and the development testbed to NetBox 4.6.5 and
   netbox-branching 1.1.1. Make the runtime guard flag any Branching mismatch.
2. Extend maintained scope-tag reconciliation to remove this sync's configured
   include-tag assignments from the current out-of-scope bucket. Resolve only
   existing tags by slug or name and never create a tag solely to remove it.
3. Automatically enqueue only bounded stale include-tag cleanup after every
   successful sync whose source enables `apply_device_scope_tags`. Keep the
   legacy explicit backfilled/out-of-scope classification automation option and
   keep destructive pruning manual.
4. Build a pure latest-sync evidence summary from persisted ingestion counters
   and snapshot IDs. Confirm convergence only for a merged, zero-change,
   zero-failure ingestion whose snapshot matches the preview.
5. Render the evidence beside the workload-only preview and include a redacted
   aggregate copy in support bundles. Export no device rows or network ID.
6. Correct operator documentation so a second preview is not described as an
   exact convergence check. The acceptance proof is a second sync against the
   same resolved snapshot with zero changes and failures.
7. Audit the repository against the Harness Engineering principles. Preserve
   the concise repository map, mechanically enforce core-knowledge freshness,
   schedule a read-only gardening signal, and record rather than hide the
   active-plan and observability gaps.

## Rollback

Revert the Python, template, package, CI, test, and documentation changes. No
schema migration or automatic deletion is introduced. Removed managed tags are
reapplied on a later in-scope upsert if Forward again reports the device with
that tag. A rollback also requires restoring the prior NetBox/Branching package
requirements together; mixed runtime versions are unsupported.

## Decision Log

- Chose 2.6.0 rather than a 2.5.12 patch because the release intentionally
  raises the NetBox floor and pins the Branching implementation.
- Rejected calling dependency workload estimates exact drift; the preview does
  not perform object-level NetBox comparisons.
- Rejected a hidden second branch/apply pass in Preview Dependencies. It would
  roughly duplicate sync cost and requires a separately designed exact-drift
  engine with explicit capacity and cleanup behavior.
- Rejected automatic orphan deletion. Durable cross-source object provenance
  does not exist yet, so Scope Reconciliation remains the reviewed authority.
- Rejected removing include-tag assignments when plugin ownership is disabled;
  those assignments may be operator-managed.
- Rejected implicitly running global backfilled/out-of-scope classification for
  every managed-scope source. Without durable object provenance it can label
  unrelated NetBox devices, so automatic behavior is limited to configured
  include-tag assignments.
- Exact pre-apply drift and durable source ownership remain explicit
  architecture work after this release, not claims attached to 2.6.0.
- Classified Harness Engineering alignment as strong but partial. The release
  adds drift detection for repository knowledge; it does not claim that 102
  active plans are healthy or that a full metrics/traces stack exists.
- Rejected relying on a local-only harness check for direct pushes. CI now runs
  the repository invariant checker for every non-Dependabot push and PR.

## Release Acceptance

1. Upgrade the staging runtime to NetBox 4.6.5 and netbox-branching 1.1.1,
   install the 2.6.0 candidate, run migrations, and restart NetBox workers.
2. Confirm `pip show forward-netbox netboxlabs-netbox-branching`, NetBox startup,
   `manage.py check`, and the plugin runtime dependency log show the exact
   matrix.
3. No query publication is required for 2.6.0 because bundled NQE is unchanged.
4. Run Preview Dependencies, then sync one explicit/resolved snapshot. Confirm
   the post-sync stale managed scope-tag cleanup completes and contradictory
   managed include/out-of-scope tags are absent.
5. Run the sync again against that same snapshot. Accept convergence only when
   the Drift Report shows zero applied, created, updated, deleted, and failed
   changes with an explicit snapshot match.
6. Export a support bundle and verify it contains the redacted latest preview,
   latest ingestion counters, and convergence state without a network ID or
   inventory rows.

## Bundled Changes

- Require NetBox 4.6.5 and pin netbox-branching 1.1.1 across package metadata,
  CI, the local testbed, runtime diagnostics, and documentation.
- Automatically clear stale plugin-managed include-tag assignments after
  successful syncs while preserving unrelated tags and avoiding implicit global
  backfilled/out-of-scope classification.
- Add persisted same-snapshot convergence evidence to the Drift Report and
  future support bundles; dependency preview remains an honest workload upper
  bound rather than false exact drift.
- Python/UI only; no NQE query change, migration, or query republish.
- Preserve agent-first harness alignment with a dated evidence map, concise
  `AGENTS.md` guard, 90-day core-knowledge freshness gate, and weekly read-only
  gardening audit.
