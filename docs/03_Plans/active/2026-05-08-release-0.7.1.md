# Release Plan: 0.7.1

## Goal

Publish `0.7.1` with the cable-ingestion remediation, transient API retry hardening, OSPF identity fallback, and density-based shard re-planning while keeping the release aligned with NetBox `4.5.9`.

## Constraints

- Keep the release line on NetBox `4.5.x`.
- Preserve the NetBox-native multi-branch workflow and branch-budget defaults.
- Do not commit customer identifiers, network IDs, snapshot IDs, credentials, or private screenshots.
- Require the repo release gate before tagging or publishing artifacts.
- Publish immutable GitHub and PyPI artifacts from the tagged commit.

## Touched Surfaces

- Package metadata in `pyproject.toml` and `forward_netbox/__init__.py`
- Release compatibility and version-history docs
- GitHub release tag and notes
- PyPI wheel and source archive

## Approach

1. Bump package and plugin version metadata to `0.7.1`.
2. Update install examples and release compatibility tables.
3. Run the repo release gate and package build checks.
4. Commit the release metadata/docs change.
5. Tag, push, create the GitHub release, and upload the PyPI artifacts.

## Validation

- `invoke ci`
- `python -m build`
- `python -m twine check dist/*`
- GitHub Actions success on `main` and the release tag

## Rollback

If validation fails before publishing, revert the release metadata/docs commit and keep `0.7.0` as latest. If publication succeeds but a release-blocking issue is found, publish a newer patch release.

## Decision Log

- Chosen: `0.7.1` patch release because the cable ingestion fix and the follow-up retry/re-planning hardening belong on the current supported line.
- Rejected: republishing `0.7.0` because GitHub and PyPI artifacts are immutable after release.
