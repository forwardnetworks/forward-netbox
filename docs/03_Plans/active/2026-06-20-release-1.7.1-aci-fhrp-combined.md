# Release 1.7.1 — ACI BD/L3Out Graduation + FHRP Churn Fix

## Goal

Ship a combined release with ACI bridge domain/L3Out NQE graduation and a
hardened FHRP churn fix, replacing the yanked 1.7.0 and 1.6.2.

## Constraints

- No customer identifiers, network IDs, or credentials in the diff.
- VIP-first lookup must not break the common FHRP apply path.

## Bundled changes

1. **ACI bridge domain and L3Out NQE maps** — promoted from `where false` stubs
   to moquery-based command-inventory queries with full field extraction.
2. **ACI VRF and tenant moquery alignment** — switched to command-inventory
   pattern for consistency; no functional change to output rows.
3. **Query publish skip-no-op** — `publish_builtin_nqe_map_queries` skips
   `edit_org_nqe_query` when committed source matches new source.
4. **Platform manifest split (create vs update)** — `_ensure_platform` uses
   separate create/update values so manufacturer is set on CREATE only.
5. **Validation audit `--overwrite` flag** — explicit opt-in to republish
   queries that differ from bundled source.
6. **FHRP shard co-location** — `row_shard_key` uses group identity so both
   HSRP routers for a group land in the same plan item.
7. **FHRP `_split_diff_rows` dedup** — DELETE rows whose coalesce identity also
   appears in upsert_rows are suppressed, preventing state-flip churn.
8. **FHRP VIP-first lookup** — `apply_ipam_fhrpgroup` finds the existing group
   via its stable VIP before falling back to name-based coalesce, preventing
   spurious create+delete cycles when NQE name data changes.
