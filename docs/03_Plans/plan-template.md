# Implementation Plan Template

## Goal

State the concrete user-visible or operator-visible outcome.

## Constraints

List compatibility, NetBox/Branching, data-safety, release, and migration constraints.

## Touched Surfaces

List the production modules, docs, tests, and scripts expected to change.

## Approach

Describe the intended behavior and the implementation sequence. Keep this decision-complete enough that another engineer can implement without guessing.

## Validation

List the exact tests, checks, Docker runs, UI verification, or release gates required.

## Rollback

Describe how to back out the change and any state that must be cleaned up.

## Decision Log

Record important alternatives rejected and why.
